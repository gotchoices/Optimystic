# Crash-failover backup promotion ↔ deterministic slot assignment (complete)

## What shipped

The TTL-failover participant path and the cohort `onRenew` were incoherent on a **crash** (no
membership rotation, `cohortEpoch` unchanged): `assignSlots` still names the dead member primary, so
a re-attach to `backups[0]` was answered `primary_moved { newPrimary: <dead primary> }` and the
participant bounced between the promoted backup and the corpse. This ticket closes that loop with:

- **Wire** — `RenewV1.reattach?: boolean`, carried inside the *signed* body (`buildRenew` includes it
  before signing) and validated via `assignDefined(out, "reattach", optBool(...))`. Backward-compatible
  (old encoder omits it → decodes as a normal ping; JSON codec round-trips it).
- **Participant** — `buildRenew(reattach)`; `reattach()` → `true`, `trySend` → `false`. `failover()`:
  `ok` → promote the contacted backup; `primary_moved` with a *different* live primary → adopt via
  `applyPrimaryMoved`; `primary_moved` pointing back at the just-failed primary → ignored (bounce
  guard); `unknown_registration`/else → next backup; all exhausted → `relookup`.
- **Cohort** — `failoverServing: Map<recordKey, epoch>` epoch-scoped override + factored
  `touchAndServe`. `reattach=true`: self==computed primary → serve; self∈computed backups & record
  held → re-stamp `primary=self`, gossip, set override, `ok`; isServing → serve; else `primary_moved`.
  Plain ping serves on computed-primary OR isServing OR current-epoch override, with housekeeping that
  drops a redundant/stale override. The re-stamp drives gossip + the next handoff, never the serve
  decision (the override does).

The implementation matched the resolved design from the source ticket; review confirmed it and fixed
two real defects (below).

### Major — none filed

### Minor — fixed in this pass

1. **`failover()` did not reset the strike count after adopting a `primary_moved`** (`renewal.ts`).
   `promote()` and the normal-ping `onPingSuccess` path both reset `consecutiveFailures = 0`, but the
   `primary_moved`-during-failover branch left it pinned at `MAX_PING_FAILURES` (3). Consequence: after
   a failover that adopts a rotated live primary, the *very next* transient failure (3+1 ≥ MAX)
   immediately re-failovers — eroding the documented "3 consecutive failures" contract for the freshly
   adopted primary. **Fix:** reset `consecutiveFailures = 0` after `applyPrimaryMoved` in `failover()`.
   **Test added:** "resets the strike count after adopting a primary_moved during failover" — adopts a
   (also-unreachable) moved primary, then asserts a single failed ping does *not* relookup and that
   relookup fires only after a fresh run of MAX_PING_FAILURES. Mutation-confirmed.

2. **`failoverServing` overrides were never cleared on eviction** (`renewal.ts`, `sweepStale`).
   The override is only cleared by the plain-ping housekeeping (on epoch change) — but an evicted
   record receives no further pings, so its override leaked for the node's lifetime. Worse than a leak:
   if the same `(topicId, participantId)` re-registers under the *unchanged* epoch (TTL lapse without a
   membership rotation), the stale override would wrongly make a non-primary member keep serving the
   new registration's plain pings. **Fix:** `failoverServing.delete(recordKey(...))` for each evicted
   record in `sweepStale`. **Test added:** "clears a failover override when its record is evicted" —
   accept re-attach → sweep → re-register under same epoch → plain ping must `primary_moved`.
   Mutation-confirmed. **Docs:** §TTL-and-renewal resolved note updated to state the override is cleared
   on epoch change *and* on TTL eviction.

### Checked, no change needed (with reasons)

- **Wire / signing.** `reattach` lives inside the signed `UnsignedRenew` body and the JSON codec
  serializes the whole object, so it round-trips and is covered by the body signature regardless of
  where verification runs. `optBool` rejects non-booleans; `assignDefined` keeps an absent field off the
  decoded object (back-compat). Round-trip / reject / absent tests present. Correct.
- **Signature verification absent from `onRenew`.** Pre-existing architectural boundary — `onRenew`
  never verified signatures (verification is db-p2p transport's job); placing `reattach` in the signed
  body is exactly what makes a future/existing verifier reject a flipped flag. Consistent with prior
  behavior; out of this layer's scope. No change.
- **`primary_moved` with no `newPrimary`.** Bounce guard only compares when `newPrimary` is defined; a
  reply lacking it falls into `applyPrimaryMoved`, which leaves `primary` unchanged (and now resets the
  strike count). Real cohort replies always carry `newPrimary`; benign defensive edge. No change.
- **Cascading failover (`self == backups[1]`, the s+2 hop).** Cohort acceptance uses
  `backups.some(self)`, so the `backups[0]`-as-self acceptance test exercises the identical branch a
  `backups[1]`-as-self re-attach would; participant-side multi-backup iteration is covered by the
  bounce-guard + unknown_registration fall-through tests. No dedicated test added — the branch is
  already exercised. (Same disposition the implementer proposed.)
- **Single-primary invariant under a revived dead primary.** Rests on the participant pinging exactly
  one member at a time (sequential `failover`, returns on first `ok`) + epoch-scoped override clearing +
  TTL staleness of the unpinged corpse. The epoch-scoped clearing test and the new eviction-clearing
  test cover the clearing side; the revived primary receives no pings and goes stale. Holds.
- **Gossip / cross-member replication is mocked** (`RecordingGossip` stub). End-to-end convergence of
  the re-stamped record onto other replicas depends on the (already-landed) gossip-membership-certs
  work and is exercised in its own suite, not re-run here. Out of this unit suite's scope. No change.
- **Delivery-level dual-serve window** (revived primary + promoted backup both pushing) is the same
  bounded window the rotation handoff already tolerates; application-level delivery dedup owns it. Out
  of this layer. No change.
