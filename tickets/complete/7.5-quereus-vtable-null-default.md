# Document Quereus SQL Dialect in Optimystic Docs

Quereus intentionally departs from standard SQL in several ways, most notably defaulting columns to NOT NULL (aligned with The Third Manifesto). The pragma `default_column_nullability = 'nullable'` is available for backward compatibility. Other differences include native temporal/JSON types, `USING` clause for virtual tables, operation-specific CHECK constraints, and all-virtual-table architecture.

These are documented in the Quereus project's `docs/sql.md` (Section 11: "Quereus vs. SQLite") but should be prominently referenced in Optimystic's own documentation so that users encounter the information without needing to navigate to the upstream project.
