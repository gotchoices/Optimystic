# Complete: Reactivity PushState gossip over the cohort bus

## Summary

`ReactivityPushStateGossipDriver` (`packages/db-p2p/src/reactivity/push-state-gossip.ts`, exported from
`reactivity/index.ts`) drives db-core's already-tested `PushState` gossip codec
(`serializeGossip`/`mergeGossip`, `encode/decodePushStateGossipV1`) over libp2p on a periodic cadence, so the
forwarder-cohort replay ring + dedupe window converge across a whole cohort. This is the **replication**
backstop to the live fan-out: a member that missed an origin dial still receives the entry via gossip and can
serve a backfill (12.4) after a primary failover, making the doc's "**any** cohort member can serve a
replay/backfill" property real over the wire.

The module is transport- and node-assembly-agnostic: it depends only on the `broadcastOver` seam
(`Pick<FretCohortGossipTransport, "broadcastOver">`) and the db-core codec, so it is unit-tested with a fake
transport and real `PushState`s. Live node wiring is the downstream `reactivity-notification-transport` (12.33)
ticket.

## Review findings

Adversarial pass over the implement commit `e30a631`. I read the implement diff (driver + spec) first, then
the db-core codec it drives (`push-state.ts`, `replay-buffer.ts`, `cohort-topic/wire/codec.ts`) and the
`broadcastOver` seam, before reading the handoff. Disposition: **no major findings**; three minor test-coverage
gaps fixed inline. Detail by aspect:

### Verified correct

- **Frame-bound binary search (`boundedFrame`/`encodeSlice`).** Monotonicity holds: `encodeSlice(g, keep)`
  keeps the most-recent `keep` ring entries (`entries.slice(len - keep)`) plus the full dedupe window, and
  JSON frame length is strictly increasing in `keep` (removing entries only removes bytes), so the
  binary-search invariant ("if `mid` fits, every smaller `keep` fits") is sound. `keep=0` ⇒ `slice(len)` ⇒
  `[]` (dedupe-floor frame); `keep>=len` ⇒ all entries (== full encode). The `kept < total` signal correctly
  drives `onTruncate` + the "no silent cap" log.
- **Encode/decode bound symmetry.** `encodeCohortMessage` bounds the **body**; `decodeCohortMessage` bounds the
  **declared body** length. The driver's `boundedFrame` compares the **full frame** (`body + 4`-byte prefix)
  against `maxBytes`, i.e. it sends *stricter* than it receives — so a self-sent in-bound frame (`frame.length
  <= maxBytes` ⇒ `body <= maxBytes - 4 < maxBytes`) always decodes. No off-by-prefix rejection. (The
  pathological dedupe-floor-overrun frame can exceed `maxBytes` and would be rejected by a peer's decode — but
  that never occurs at the 512 KiB production bound, and is surfaced via `onTruncate`/log; documented in the
  handoff. Acceptable.)
- **Merge safety under clipping.** `ReplayBuffer.merge` is a per-revision union with a `receivedAt` tie-break
  (`replay-buffer.ts:107`); same-revision payloads carry the same end-to-end signature, so a clipped
  most-recent-N frame loses nothing a peer already holds and cannot be forged into a peer's ring. Clipping to
  the high (newest) end is the right slice — a lagging subscriber backfills the newest gap first.
- **`deliver` never-throw contract + gate ordering.** decode (try/catch → drop+log) → `isCohortMember` gate
  (drop non-member before resolve/merge) → resolve (`undefined` ⇒ ignore) → `mergeGossip` (try/catch). The
  resolve step is a fast pre-filter; `mergeGossip` independently re-guards collectionId/topicId
  (`push-state.ts:189`), so a mis-routing resolver cannot corrupt a foreign state (defense in depth, tested).
- **Timer discipline parity with the host tick.** `stopped`/`rounding` re-entrancy guards, idempotent `start`
  (no second timer), `unref()` so an idle process is never pinned, `stop()` short-circuit + `clearInterval`.
- **Type safety.** `Pick<FretCohortGossipTransport, "broadcastOver">` matches the seam's
  `(protocol, coord, frame)` signature exactly; `RingCoord`/`PushStateGossipV1` imports are type-only.
- **Resource cleanup.** Single `unref`'d interval, cleared and nulled on `stop()`; no other handles.

### Minor findings — fixed inline (in this pass)

The handoff *claimed* per-collection isolation, multi-collection iteration, and a mid-loop `stopped` break, but
the 9 shipped specs only ever exercised a **single** live collection and never the round's error path. Added 2
specs (`packages/db-p2p/test/reactivity/push-state-gossip.spec.ts`) to lock these in:

1. **`broadcasts every live collection in a round and isolates a per-collection failure`** — a 3-collection
   round where the middle collection's `broadcastOver` throws; asserts the round does not throw and the two
   healthy collections still broadcast (exercises the for-loop over multiple collections + the per-collection
   `try/catch`).
2. **`stops mid-round when stop() lands during iteration (the stopped break)`** — a generator `liveCollections`
   that calls `driver.stop()` between yields; asserts only the pre-stop collection broadcast (exercises the
   mid-loop `if (this.stopped) break`).

Spec now 11 passing (was 9).

### Considered, intentionally not changed (no defect)

- **Node wiring / live timer absent.** By design — 12.33 step 5. The driver is inert until 12.33 constructs and
  `start()`s it. Not a gap in *this* ticket.
- **Host accessor + `cohortCoord` threading.** `ReactivityForwarderHost` exposes no live-collection iterator
  nor a `(collectionId,topicId)→PushState` lookup, and `PushStateInit` carries no `RingCoord`. These are
  12.33's `liveCollections`/`pushStateForGossip`/cohortCoord wiring concerns, already flagged in the handoff
  (a minimal `servedTopics()` accessor on the host is the clean addition). Correctly left out of the
  driver-only scope.
- **Envelope signing.** Deferred; backlog ticket `reactivity-pushstate-gossip-envelope-signing.md` exists.
  PushState-gossip frames are authenticated by the `isCohortMember` gate, and every replay entry is a full
  end-to-end-verifiable `NotificationV1` that a backfill *serve* re-verifies regardless — a poisoned ring entry
  can only waste a slot, never forge a delivery. Acceptable interim posture.
- **Most-recent-N convergence limit.** Convergence to a member's full ring requires the per-round ingest delta
  to fit one frame; a larger burst leaves older entries un-replicated by that member's own gossip, covered by
  (a) other members holding them and (b) checkpoint/backfill (12.4). Consistent with the doc's layering;
  rotating-window truncation is a possible future enhancement, not a defect.
- **`deliver` does not check `stopped`.** A post-`stop()` inbound frame still merges; harmless (idempotent
  union into a state being torn down) and not worth a guard.
- **`clock` dep unused.** Accepted for parity with the host tick; gossip frames carry their own `receivedAt`.
