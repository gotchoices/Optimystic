## What was built

**RebalanceMonitor** — a `Startable` service that detects topology changes (libp2p `connection:open`/`connection:close`) and identifies responsibility shifts for locally-tracked blocks using FRET's `assembleCohort()` + `hashKey()`.

Key behaviors: debounced topology change handling (default 5s), throttled rebalance scans (default 60s), partition suppression via `PartitionDetector`, `RebalanceEvent` emission with gained/lost blocks and new owners.

## Review findings & fixes

- **DRY fix**: `RebalanceMonitor.setStatus` was duplicating the logic already in `ArachnodeFretAdapter.setStatus`. Refactored to delegate directly.
- **Performance fix**: Hoisted `TextEncoder` allocation from per-block-per-check loop to module-level constant.
- **Test cleanup**: Simplified the `setStatus` test to verify delegation rather than re-implementing adapter logic in a mock.

## Usage

```typescript
const monitor = networkManager.initRebalanceMonitor(partitionDetector, fretAdapter, {
  debounceMs: 5000,
  minRebalanceIntervalMs: 60000,
  suppressDuringPartition: true
});
monitor.onRebalance(event => coordinator.handleRebalanceEvent(event));
monitor.trackBlock('block-abc');
await monitor.start();
const event = await monitor.checkNow();
monitor.setStatus('moving');
```
