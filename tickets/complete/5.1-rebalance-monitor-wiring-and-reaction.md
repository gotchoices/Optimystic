# Complete: Wire RebalanceMonitor + react via BlockTransferCoordinator on a live node

## What landed

The rebalance path is now activated end-to-end on a real node. Inside the arachnode/FRET gate of
`createLibp2pNodeBase` (`packages/db-p2p/src/libp2p-node-base.ts`):

- constructs a `BlockTransferCoordinator` against the **local** `storageRepo`, `keyNetwork`, the
  `RestorationCoordinator`, the `partitionDetector`, and the `/optimystic/<networkName>` prefix that
  matches the registered block-transfer receive handler;
- inits + starts the `RebalanceMonitor` via `networkManager.initRebalanceMonitor`;
- hops the monitor's `onRebalance` event into `coordinator.handleRebalanceEvent` (pull gained / push
  lost, each partition-guarded);
- feeds owned blocks from `storageRepo.onAnyCollectionChange` into `monitor.trackBlock`;
- exposes `node.rebalanceMonitor` / `node.blockTransferCoordinator`, composes a `node.stop` wrapper
  that releases the subscription and stops the monitor, and wraps the whole block in a non-fatal
  `try/catch` (rebalance is a resilience optimization, not a correctness requirement).

New `NodeOptions.rebalance?: Partial<RebalanceMonitorConfig> & { enabled?: boolean }` gates the path
(default on). When arachnode is disabled or FRET is absent the path stays inert by design.

## Review findings

### Scope / what was checked
Read the full implement diff (`03974ab`) with fresh eyes before the handoff summary, then cross-checked
against the source of `block-transfer.ts`, `rebalance-monitor.ts`, `network-manager-service.ts`,
`restoration-coordinator-v2.ts`, and the spread block in `libp2p-node-base.ts` it mirrors. Verified:
in-scope variables (`keyNetwork`/`partitionDetector`/`protocolPrefix`/`networkManager` all defined
before the block); protocol-prefix match with the receive handler; local-`storageRepo` choice;
`initRebalanceMonitor` signature; `node.stop` wrapper composition order (clearInterval → rebalance →
spread → base, each calling its captured previousStop in `finally`); idempotency of double-stop
(`NetworkManagerService.stop()` + the wrapper both call `monitor.stop()`, which early-returns when not
running); and the reaction/test logic.

### Finding — error handling (MINOR, fixed inline)
The `onRebalance` hop was `void coordinator.handleRebalanceEvent(event)`. `handleRebalanceEvent` **can
reject**: `executePull` awaits `RestorationCoordinator.restore()` inside a `try/finally` with **no
catch**, and `restore()` itself awaits `getMyRingPeers()`/`queryPeer()` without a guaranteed catch. A
bare `void` would surface such a rejection as an **unhandled promise rejection** — process-fatal on
Node ≥15 — defeating the deliberate non-fatal `try/catch` around node startup. Fixed by swallowing +
logging the rejection: `coordinator.handleRebalanceEvent(event).catch(err => log?.(...))`. Updated the
matching hop in `rebalance-reaction.spec.ts` (and its comment) so the test stays a faithful mirror of
what the node-base installs.

### Findings — accepted as-is (with reasons)
- **`enabled` excess property forwarded to `initRebalanceMonitor`.** `options.rebalance` (incl.
  `enabled`) is passed wholesale; `RebalanceMonitorConfig` ignores `enabled`. Harmless, and
  **intentionally consistent** with how the spread block forwards `options.spreadOnChurn` wholesale —
  left to avoid diverging the two parallel paths.
- **Two owned-block subscriptions / tracked sets (spread + rebalance).** A second
  `onAnyCollectionChange` subscription and a per-monitor tracked `Set`. Deliberate; collapsing onto one
  shared set + eviction from the `lost` signal is the already-filed follow-on
  `unify-monitor-tracked-block-set`.
- **Initial-scan gap.** Blocks already durable from a previous run are not re-emitted on startup, so
  they are untracked until next touched — same inherited caveat the spread block documents; deferred to
  `optimystic-owned-block-initial-scan-seed`.
- **No live socket-level rebalance E2E.** The unit + real-libp2p wiring specs are the floor; a
  monitor-driven block move over real sockets on a multi-node mesh is integration-scale and not added
  here. The reaction primitive is socket-tested for block-transfer/spread elsewhere. Acceptable;
  candidate for an env-gated integration test if desired — **not** filed as a blocking ticket.
- **Test hygiene.** `rebalance-monitor-node-wiring.spec.ts`'s "node.stop() stops the monitor" case
  does not wrap in `try/finally` (the other three cases do); on an assertion failure before
  `node.stop()` the node would leak. Cosmetic, test-only, no production impact — left.
