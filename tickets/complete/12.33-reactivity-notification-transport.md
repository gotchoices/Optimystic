# Reactivity notification transport — live node wiring (COMPLETE)

## Summary

The `cohortTopic`-enabled production node (`libp2p-node-base.ts`, the `if (cohortEnabled)` block) composes the
full reactivity transport stack — notify transport, forwarder host, origination emit, node-level subscriber
registry, and push-state gossip — so a committed change on a tail-cohort member fans out over the real
`/optimystic/reactivity/1.0.0/notify` socket to remote subscribers, and inbound frames route by topic to both
the forwarder role (`onInbound`) and the subscriber role (`ReactivitySubscriberRegistry`). The pinned
`BlockId → tail-bytes` encoding moved to a single source of truth (`reactivity/topic-bytes.ts`), re-exported by
the membership gate for back-compat. The cohort-topic host now exposes `profile` and `gossipTransport` so the
reactivity layer reuses them instead of standing up duplicate state.

The implementation is correct, builds clean, and is validated end-to-end on real libp2p. See review findings.
