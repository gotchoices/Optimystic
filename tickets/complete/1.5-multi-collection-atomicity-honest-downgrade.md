# Complete: multi-collection atomicity — honest downgrade + reporting-contract tests

## Summary

Optimystic commits a multi-collection transaction **collection-by-collection with no
cross-collection undo**, so a mid-loop failure can half-land. The plan settled on
**Option B — honest downgrade** (atomicity of *intent* + eventual, *reported* visibility)
over real cross-collection 2PC, because the trigger is a *permanent stale loss* (a racing
transaction steals a collection's log-tail slot between its PEND and COMMIT) that a 2PC
journal cannot heal. Option A is parked as backlog `feat-cross-collection-atomic-commit`.

The implement stage did the doc downgrade + reporting-contract tests; **no runtime behavior
was changed** (the machinery — `CoordinatorPartialCommitError`,
`ExecutionResult.committedCollections/failedCollections`, the split commit/execute partial
paths — shipped under the earlier `txn-multi-collection-commit-honest-reporting` ticket).

## Contract the tests pin (unchanged from implement)

- A multi-collection `commit()` that half-lands throws `CoordinatorPartialCommitError` naming
  the durable set vs the reverted set; winner gets fold+reset+clear, loser is restored for clean
  retry, `stampData` is dropped.
- `execute()` surfaces the same partition on `ExecutionResult` **without throwing**
  (non-retryable, not snapshot-wrapped), applying `recordCommitted`+`tracker.reset` to the
  committed subset only.
- A clean failure (PEND fails, or every collection loses) throws a **plain** `Error`, never the
  partial signal, leaving every tracker pristine.
- Re-driving `commit()` after a partial landing re-attempts **only** the failed collection — no
  duplicate log entry on the winner.
