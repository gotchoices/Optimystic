## What was built

`RingSelector.determineRing()` (`packages/db-p2p/src/storage/ring-selector.ts:37`) now consults `ArachnodeFretAdapter.getRingStats()` to compute the network's observed total spare capacity as `Σ (peerCount × avgCapacity)` across known rings, replacing the prior hard-coded `100KB × 1000` constant. Ring depth then comes from the existing `ceil(-log2(coverage))` math where `coverage = capacity.available / estimatedTotalData`.

Three behavioral branches:
1. **Populated stats** — sum across all observed rings becomes the demand estimate.
2. **Empty stats (bootstrap)** — falls back to the prior `100MB` constant so first-boot behavior is unchanged.
3. **Pathological zero** — if the aggregate is `<= 0` (e.g. all peers report zero available), fall back to `capacity.available` so coverage = 1.0 and the node claims ring 0.

The constructor field was renamed from `_fretAdapter` to `fretAdapter` (it is now actually consumed). `shouldTransition()` was not modified — it routes through `determineRing()` and benefits transparently. `libp2p-node-base.ts:387` was already wiring a real adapter, so no signature change rippled.

A `docs/arachnode.md` §"Capacity Management and Ring Adjustment" reference was added to the in-code comment to anchor the design rationale.

## Use cases / scenarios

1. **Cold start** (no peers visible): bootstrap fallback (~100MB) preserves prior behavior; new node picks a sensible default ring without needing FRET populated.
2. **Established network, single ring**: ring 0 with N peers averaging A available; new node with `a` available picks `ceil(-log2(a / (N×A)))`. Larger N → deeper ring → finer partition.
3. **Multi-ring network**: aggregation includes capacity at all rings, so a node with modest capacity correctly chooses a deep ring even when the network is partitioned.
4. **Capacity pressure**: as `capacity.used` rises, `available` falls, coverage shrinks, deeper ring; `shouldTransition()` (`usedPercent > moveOut`) signals 'out'.
5. **Network shrinks**: peers leave, aggregate falls, coverage grows, shallower ring on next sample.
