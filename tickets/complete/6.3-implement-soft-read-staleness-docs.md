## What shipped

Pure-docs ticket. Added `## Read Consistency and Staleness` to `docs/transactions.md`
documenting two staleness sources — non-responsible **soft serves** and the **lazy
read-repair window** — the config knobs, the worst-case bound, and the
transaction-vs-bare-read distinction (transaction read-dependencies are validated at
commit, catching stale data; bare reads are not). A callout was added to the
plugin README cross-linking the section, and a reference-only bullet was added to
`right-is-right.md` Open Design Questions pointing at the gossip/reputation backlog.
