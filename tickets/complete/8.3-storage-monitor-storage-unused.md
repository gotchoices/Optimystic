## What was built

`StorageMonitor.estimateUsedSpace()` previously returned `0`; it now delegates to
`this.storage.getApproximateBytesUsed?.() ?? 0`, restoring the storage field from
a lint-silenced `_storage` to a live `private readonly storage`.

`IRawStorage` gained an optional `getApproximateBytesUsed?(): Promise<number>`.
Optional preserves source compatibility with out-of-tree implementers; `StorageMonitor`
defends with `?.()` and `?? 0`.

`getCapacity()` override gating was tightened: previously a bare `totalBytes`
config (a disk-size hint) put the monitor entirely into override mode and forced
`used = 0`. Now used/available overrides are honored independently of `totalBytes`,
so `totalBytes` alone caps the disk size while the backend still reports actual
used bytes — the production-meaningful path that was previously broken.

### Backend implementations

- **FileRawStorage**: recursive `fs.readdir(..., {withFileTypes:true})` + `fs.stat(file)`
  walk over `basePath`, summing file sizes. ENOENT swallowed (concurrent deletion
  or fresh node before `basePath` exists). Acceptable as-is — invoked only on
  `shouldTransition()` checkpoints (every 60s in `libp2p-node-base.ts:420`).
- **MemoryRawStorage**: sums `key.length + jsonByteSize(value)` across its five
  maps; an approximation, but a usable proxy for capacity ratios.
- **MMKVRawStorage**: iterates `mmkv.getAllKeys()` filtered by `this.prefix`,
  summing `key.length + storedString.length`.

### Wiring (unchanged)

`libp2p-node-base.ts:386` already passed the live `rawStorage` into
`new StorageMonitor(...)`; no call-site change needed.

## Tests

`packages/db-p2p/test/storage-monitor.spec.ts` — six tests:
1. Reflects backend-reported bytes when no overrides are supplied.
2. Treats missing `getApproximateBytesUsed` as 0 — `available === total`.
3. Clamps reported used to `total` when the backend overshoots.
4. Floors negative reports at 0.
5. `usedBytes` override wins; the backend is not called.
6. `availableBytes` override yields `used = total - available`.

## Review notes

- FS recursive walk per call is acceptable: callers hit `getCapacity()` only on
  ring-transition checkpoints (60s interval). A TTL cache would be a follow-up
  optimization, not blocking.
- Memory backend's JSON-length proxy is acceptable; the only consumer needs a
  ratio, not byte-exactness.
- Override gating semantics confirmed: `totalBytes` is a disk-size hint and must
  not suppress backend reads.
- `?.()` error semantics preserved: only short-circuits on `undefined`; thrown
  errors propagate as before.

## Usage

Consumers configuring a node need no API changes — `libp2p-node-base` constructs
`StorageMonitor(rawStorage, options.arachnode?.storage ?? {})` and the wiring is
transparent. To override at runtime:

```ts
new StorageMonitor(rawStorage, {
  totalBytes: 50 * 1024 * 1024 * 1024, // 50GB cap; backend still reports used
});

new StorageMonitor(rawStorage, {
  totalBytes: 50 * 1024 * 1024 * 1024,
  usedBytes: 10 * 1024 * 1024 * 1024,  // explicit override; backend not called
});
```

Out-of-tree `IRawStorage` implementers without `getApproximateBytesUsed` degrade
gracefully to `used = 0` (same as pre-fix behavior — no transitions, but no crash).
