# Complete: hoist duplicated wire-validation primitives into one shared module

## What this was

The generic structural-validation helpers (`asObject`, `requireV1`, `reqString`, `reqFiniteNumber`,
`reqStringArray`, the base64url decoders, the fail-with-`CohortWireError` shim, etc.) were copy-pasted
across the cohort-topic, matchmaking, and reactivity wire codecs and had begun to diverge. The implement
stage consolidated them into `packages/db-core/src/cohort-topic/wire/primitives.ts`, which every codec now
imports. `CohortWireError` moved into `primitives.ts` and is re-exported from `validate.ts` so existing
importers are unaffected. Behavior-preserving consolidation only — the one behavior change
(`b64urlFixedLen` byte-width check) landed in the earlier `cohort-topic-wire-hardening` ticket and is
preserved here.

## Review findings

Adversarial pass over commit `7ad74e5`. The change is a behavior-preserving refactor; the review focused
on the reconciliation of the previously-divergent copies (the only place a bug could hide) and on whether
any consumer was left referencing a moved/renamed symbol.

**Correctness of the primitive reconciliation — checked, no defects.**
- The 9 migrated `reqIntInRange` call sites in `matchmaking/wire.ts` flipped from the value-form
  `reqIntInRange(reqFiniteNumber(obj, key, what), key, what, min)` to the obj-form
  `reqIntInRange(obj, key, what, min)`. The obj-form calls `reqFiniteNumber(obj, key, what)` internally, so
  each is byte-for-byte equivalent — verified the field key, min, and max match at every site (incl.
  `limit → 1..QUERY_LIMIT_MAX`, the `minBudget` optional branch in `validateCapabilityFilter`, and the three
  `AggregateBucketV1` buckets). The `minBudget` branch double-reads `obj["minBudget"]` (once via
  `optFiniteNumber`, once inside the obj-form) — harmless on a `JSON.parse` object with no getters.
- `reqStringArray` obj-form migration on reactivity `NotificationV1.signers` (`wire.ts:182`): old
  `reqStringArray(obj["signers"], …)` vs new `reqStringArray(obj, "signers", …)` read the same field and
  apply the same `Array.isArray` + per-element string check. `signers` is required, so the absent-field
  path (throws) is unchanged.
- `reqBoolean → reqBool` rename on reactivity `NotificationV1.invalidation` — same body, canonical paired
  name adopted.
- Error-message strings are byte-identical across the old copies, so no message-asserting test can observe
  the move.

**Single-sourcing — verified.** After the change, every primitive (`asObject`/`req*`/`opt*`/`b64url*`/
`reqEnum`/`failWire`/`class CohortWireError`) is defined **only** in `primitives.ts` (grep sweep of
`db-core/src`). The lone surviving local helper is `reqInt` in `push-state.ts` — a range-free integer check
with no lower bound, genuinely distinct from the shared `reqIntInRange` (which requires a `min`); folding it
in would add a `>= 0` bound and change behavior. Correct to keep local; documented at its definition site.
Repo-wide grep confirms nothing outside `db-core` references the removed/renamed helpers (`reqBoolean`,
matchmaking's non-exported locals) and no stale importer of the old symbols survives.

**Import cycle — checked, safe.** `primitives.ts → codec.ts → validate.ts → primitives.ts`. All cross-edges
are runtime-only (a class + functions used inside function bodies, never at module-init), and the
`validate ↔ codec` edge pre-dates this ticket. No init-order hazard.

**Minor — fixed inline.** `matchmaking/wire.ts` dropped its `CohortWireError` import but a JSDoc
`{@link CohortWireError}` (line 275) still referenced it, leaving a dangling editor-hover link (cosmetic;
`tsc` does not flag it with `checkJs` off). Rewrote the link to the `import("…/primitives.js").CohortWireError`
form so it resolves without adding an unused value import. Build re-verified clean after the edit.

**Test coverage — assessed, floor is adequate.** The flagged high-risk `reqIntInRange` fields already have
negative tests in `test/matchmaking/wire.spec.ts` (negative `capacityBudget`, `wantCount < 1`, out-of-range
`limit`), and those tests import `CohortWireError` via the `validate.ts` re-export, exercising the relocated
symbol. Not every numeric field has an explicit out-of-range case, but the migration is provably
behavior-identical, so exhaustive per-field rejection tests are not warranted for this consolidation. No
new tests added.

## Tripwires (parked, not tickets)

- **`reqInt` range-free boundary** — code comment at its definition in `push-state.ts`. If a second consumer
  ever needs a range-free integer check, promote it to `primitives.ts`; a single local helper is not a
  "duplicated set." (Recorded by the implementer; verified present.)
- **`b64urlField` has no max-length ceiling** — `NOTE:` comment already at the definition in `primitives.ts`
  (variable-width fields can bloat a map key under a hostile peer; add `b64urlMaxLen` only if observed in
  practice). Inherited from `cohort-topic-wire-hardening`; unchanged.
- **`rotation.ts:205` `{@link CohortWireError}`** — pre-existing dangling JSDoc link (rotation imports from
  `./wire-validate.js` but not the class). NOT a regression from this ticket; the new `export *` in
  `wire-validate.ts` actually makes it resolve now. Left as-is — noting it here so a future reader doesn't
  attribute it to this change.
