### Minor — fixed inline this pass

  analysis the `unwillingFraction` edge actually reflects whichever sub-condition flips first — at
  the defaults the `≤ d_max+2` **hop bound**, not give-ups (those need `f≈1`). The `unwillingBreach`
  diagnostic captured this, but only on the in-memory report — it was **not** folded into the
  `Metrics` sink, so the JSON/CSV export (the artifact `fold-simulator-findings-into-design-docs`
  consumes) would read the `no-give-ups` claim edge as a give-up edge. Fixed: `runTreeBoundaries`
  now records `boundary.unwillingBreach` keyed by `(axis, breach)` so the export self-documents which
  sub-condition the edge reflects. Added a test asserting the fold; full suite re-run green
  (244 → was 243).
