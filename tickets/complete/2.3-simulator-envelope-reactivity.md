# Complete: reactivity operating-envelope boundaries

Two reactivity operating-envelope boundaries on the generic `findBoundary` harness:

1. **`revision-continuity` × commit rate `cps`** (pure function) — a reconnecting subscriber must
   replay from inside the layered recovery window `W + W_checkpoint` (= 4352 revisions). A 60 s
   reconnect gap at `cps` commits/sec is a lag of `round(cps·60)` revisions; `holds` is "`classifyResume`
   is not `OutOfWindow`". Located edge **`cps* ≈ 72.5`** vs nominal 10 (margin +62.5). The conservative
   replay-only bound (`W/gap ≈ 4.27`) is reported alongside and sits *below* nominal — the adaptive-`W`
   finding.
2. **`completes-within-drain` × `T_rejoin_jitter / T_drain` ratio** (seeded burst) — on tail rotation
   the subscriber set re-registers at the new tail jittered over `T_rejoin_jitter` while the old tail
   forwards until `T_drain`. `holds` maps the ratio to `T_rejoin_jitter` and runs the seeded
   `simulateRotationBurst`. Located edge **`ratio* ≈ 1.0`** vs shipped 0.5 (margin +0.5).

Both fold into a `Metrics` sink via `recordBoundary`, keyed by `(claim, axis)`. New surface exported
from `index.ts`; README envelope-finder section documents `runReactivityBoundaries` (replay-window /
rotation-drain axes).

### Correctness — checked, no bugs found
- **Monotone-in-harm holds on both axes (the load-bearing harness assumption).** Continuity:
  `round(cps·gap)` is non-decreasing in `cps`, so `holds` flips true→false once and never back —
  confirmed `monotoneViolated === false`. Drain: verified against `rng.ts` that
  `nextInt(b)=⌊nextFloat()·b⌋` reuses the same per-draw float (the fork is seeded by `(seed, label)`
  only, independent of draw order), so each arrival `⌊f_i·b⌋` and hence the max is non-decreasing in
  `T_rejoin_jitter = round(ratio·T_drain)` — `monotoneViolated === false` confirmed.
- **`simulateRotationBurst` horizon change is side-effect-free for all callers.** Enumerated every
  caller (`find_references`): `boundary-reactivity.ts`, `TailRotationScenario` (`scenarios.ts`), and
  `reactivity.spec.ts` — all use the default config (`T_rejoin_jitter` 30 s < `T_drain` 60 s), so
  `max(T_drain, T_rejoin_jitter) === T_drain` and the default burst is byte-for-byte unchanged. Both
  `reactivity.spec.ts` and `scenarios.spec.ts` stay green, as predicted.
- **Floor/edge guards.** `evalContinuity`/`coverageSeconds` throw `RangeError` on `cps ≤ 0`; the axis
  `lo = 1` so the scan never probes a silent (0 cps) topic. `CONTINUITY_BASE_REVISION = 1e6` exceeds
  any modeled gap lag (ceiling 1000 cps × 60 s = 60 000), so `currentRevision − lag > 0` always.
- **Determinism + non-vacuity.** Report is byte-identical across two runs (seed-fixed); the drain
  burst varies across seeds (so determinism is reproducibility, not a seed-independent constant).
  Positive controls flip both claims (continuity `holds(1000)===false`, 100 cps ⇒ `OutOfWindow`; drain
  `holds(2.0)===false`, last arrival past edge > `T_drain`), and the fan-out witness
  (`peakRootDirect === cap_promote_fast && finalDepth > 0`) proves the drain claim is held up by the
  fast-promote mechanism, not a trivially-short window.
- **Doc alignment.** `docs/reactivity.md` confirms every constant the boundaries assume: `W=256`,
  `W_checkpoint=4096`, stacked range 4352, 60 s recovery floor, fixed-`W` dropping below floor at
  ≥ 10 cps (adaptive-`W`), `T_drain=60 s`/`T_rejoin_jitter=30 s`, and the 29,995 ms worked-example
  last arrival ≤ `T_drain`.
