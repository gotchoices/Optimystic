# Summary

V1 dispute escalation protocol for the Right-is-Right architecture. When a transaction proceeds despite minority rejections (threshold-promises), the overridden minority can escalate to independent arbitrators selected by FRET ring-distance. Arbitrators re-execute the transaction and vote; a super-majority determines whether the challenger or the majority was right. Reputation penalties (FalseApproval=40, DisputeLost=30) are applied to the losing side.

This is V1 (async disputes — transaction commits first, then dispute runs). The target architecture (docs/right-is-right.md) calls for synchronous blocking, cascading escalation through a dissent coordinator, and ejection of the losing side.

# Review Findings & Fixes

## Bug Fixed: `getDisputeStatus` for resolved disputes

`findChallengeForDispute` only searched `activeDisputes`, but challenges are removed from that map upon resolution. Resolved disputes could never be found by `getDisputeStatus`. Fixed by adding a `resolvedChallenges` map that retains challenges after resolution. Two new tests added covering `committed-invalidated` and `committed-validated` status lookups.

## Notes for Future Work

- `disputedTransactions` Set and `resolvedDisputes`/`resolvedChallenges` maps grow unbounded — pruning should be added when dispute volume is expected to increase.
- `disputeProtocolService` (the libp2p stream handler in service.ts) is imported in libp2p-node-base.ts but not wired into the node's service registry. The mock transport in tests masks this. Will need wiring when dispute is used in production.
- The architecture doc describes synchronous blocking disputes with cascading escalation and peer ejection — significant V2 work remains.

# Usage

Opt-in via `disputeEnabled: true` on `ClusterConsensusConfig`. A `ClusterMember` whose rejection is overridden calls `DisputeService.initiateDispute(record, evidence)` to trigger arbitration. Results are broadcast to all cluster peers and arbitrators. Engine health monitoring auto-throttles nodes that repeatedly lose disputes.
