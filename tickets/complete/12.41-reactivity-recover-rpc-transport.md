# Reactivity recover RPC transport — completed (reviewed)

## Summary

The db-p2p transport for the reactivity **recovery (pull)** direction — backfill + resume over one
request-reply libp2p protocol (`/optimystic/reactivity/1.0.0/recover`). A subscriber that detected a gap
(or woke from sleep) asks a serving cohort member "what did I miss?" and is brought current. Delivered:

- **db-core wire foundation** (materialized here out of necessity — see *Cross-ticket note*): `recover.ts`
  discriminated `RecoverRequestV1`/`RecoverReplyV1` envelope + codecs; `BackfillV1.timestamp` freshness
  field; canonical `backfillSigningPayload`/`resumeSigningPayload` (type-tagged, explicitly-ordered JSON
  arrays); `createBackfillRequester` `clock` dep stamping the signed timestamp.
- **db-p2p transport machinery**: `Libp2pReactivityRecoverTransport` (outbound — sticky-primary-first then
  cohort-walk target selection, self never dialed, injected `RecoverDialer` seam);
  `decodeCohortHintTarget` encoding bridge; `createRecoverRequestHandler`/`registerRecoverHandler` (inbound
  serve — bounded decode → peer-sig verify against the **dialing** peer → node-level replay guard keyed on
  signature bytes + timestamp → resolve live `PushState` → `serveBackfill`/`serveResume` → reply, with
  no-reply on any failure); `forwarder-host.pushStateForCollection`; subscription-manager backfill
  escalation (`backfillMaxRetries` → `resume()` → chain-read) and real-`subscriberCoord` plumbing.
- **Live node wiring is intentionally NOT here** — split into `reactivity-recover-node-wiring` (in
  `implement/`, prereq `reactivity-notification-transport`), because the composition sits inside the
  reactivity block of `libp2p-node-base.ts` that `reactivity-notification-transport` (12.33) owns and which
  is currently absent (12.33 errored mid-flight). The transport + handler are injectable units precisely so
  that wiring is a thin composition step once 12.33 lands.

**Correctness / security — checked, no defects found.**
- Signing/verification contract is consistent end-to-end: requester signs over the full unsigned image incl.
  `timestamp`; handler verifies `backfillSigningPayload`/`resumeSigningPayload` against the connection's
  `remotePeer` (the dialer *is* the signer); replay guard keys on the signature bytes (verify-then-admit
  order is correct — replay state is never recorded for an unverified request). The forged-fresh-timestamp
  attack is closed (timestamp is in the signed image — pinned by an Ed25519 test).
- Target selection: sticky primary decoded via `decodeCohortHintTarget` (the load-bearing
  base64url-of-bytes → peer-id-string bridge, pinned both ways), then cohort-walk; self excluded; dups
  collapse; a malformed sticky primary is logged and skipped to the walk. Encoding round-trip verified
  against `peerIdToBytes`/`bytesToPeerIdString`/`peerIdFromString`.
- No-reply-on-failure invariant holds: decode/verify/replay/foreign-collection/no-`PushState` all return
  `undefined` and never throw out of `handleRequestResponse`'s callback.
- Adding `recover` to `reactivityProtocolList` creates **no** dangling `node.unhandle` today —
  `reactivityProtocolList` is currently referenced only from a test; the live teardown that will enumerate
  it is part of the deferred node wiring, so the list and the handler register/unregister land together.

**Minor observations — judged acceptable, NOT changed (documented for the next agent):**
- `Libp2pReactivityRecoverTransport.exchange` treats a *successful dial* that returns an undecodable or
  wrong-`kind` reply as **terminal** (it does not fall through to the next target — only dial failures do).
  This is a defensible "a valid cohort member returning garbage is a protocol error, not a retryable dial
  failure" stance and is documented in-code. Left as-is.
- The `reply.backfillReply === undefined` / `reply.resumeReply === undefined` throws in the outbound
  transport are effectively **unreachable** (the decoder's `rejectStrayBranch` + per-kind validation
  guarantee the populated branch matches `kind`). Harmless defensive guards against the optional typing;
  left as-is.
- Backfill-escalation retry re-stamps `timestamp` from `Date.now()`, so the retried request gets a fresh
  signature and is not self-rejected by a server's replay guard — *except* for two attempts inside the same
  millisecond, an edge that only arises after a transport failure (meaning the server never recorded the
  first), so it degrades at worst to a resume escalation. Acceptable.
