## What shipped

- **`'mesh-test'` factory path** in `CollectionFactory.createTransactor`:
  `createMesh(1, …) + buildNetworkTransactor(mesh)` → full production
  `StorageRepo → CoordinatorRepo → NetworkTransactor` chain. `'test'`
  (StorageRepo pass-through) preserved for specs that explicitly want it.

- **Dead-plumbing fix**: `parseTableSchema` now reads
  `tableSchema.vtabAuxData` as a fallback below explicit `USING optimystic(...)`
  args (`args['transactor'] ?? aux['default_transactor']`). Plugin-level
  `register(db, { default_transactor, default_key_network, default_port,
  default_network_name })` now actually flows through. Previously silent —
  every spec that "used" `'test'` via `default_transactor` was booting
  `createLibp2pNode` via the default `'network'` path.

- **Harness relocation**: `packages/db-p2p/test/mesh-harness.ts` →
  `packages/db-p2p/src/testing/mesh-harness.ts` + `index.ts` barrel. New
  `"./testing"` subpath export in `packages/db-p2p/package.json`
  (`./dist/src/testing/*`). Plugin imports via
  `import { createMesh, buildNetworkTransactor } from '@optimystic/db-p2p/testing'`.
  All 7 in-package specs updated; old `test/mesh-harness.ts` deleted.

- **Spec migrations**: `schema-support`, `engine-execution`, `schema-migration`
  (×2) now run through `'mesh-test'`. `distributed-quereus.spec.ts` was
  intentionally **not** migrated — it drives real libp2p with explicit
  `transactor='network'` per-table; migrating contradicts its purpose.

- **Sanity spec**: `mesh-test-transactor.spec.ts` asserts DDL + DML round-trip
  with a 5s per-test timeout to fail fast on hangs.

## Usage

Plugin specs that want the full production stack over a mock mesh:

```ts
const plugin = register(db, {
    default_transactor: 'mesh-test',
    default_key_network: 'test',
    enable_cache: false,
});
```

Or explicit per-table:

```sql
CREATE TABLE t (id INTEGER PRIMARY KEY)
USING optimystic('tree://x/t', transactor='mesh-test');
```

Specs that only need StorageRepo pass-through (no coordinator/transactor
behavior exercised) should keep `default_transactor: 'test'`.
