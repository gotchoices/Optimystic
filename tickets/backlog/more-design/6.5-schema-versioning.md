# Schema Versioning for Multi-Node

description: no explicit schema version tracking; schema cache can become stale when another node updates the schema
prereq: none
files:
  - packages/quereus-plugin-optimystic/src/schema/schema-manager.ts
----

The new Lens system in Quereus v4 should allow us to elegantly handle schema versioning.  It gives a mechanism for multiple versions of an app to co-exist simultaneously during a distributed migration with data converted both ways.  The parts we need to fill in are around how distributed authorization of schema versions works within an engine.
