# Review: reactivity operating-envelope boundaries

Two operating-envelope boundaries for reactivity were implemented on the generic `findBoundary`
harness, each an axis + a predicate over an existing reactivity point-claim
(`TailRotationScenario`'s `revision-continuity` and `completes-within-drain`). Build is green (strict
`tsc`), full package suite is **238 passing** — the 11 new specs live in `boundary-reactivity.spec.ts`
(4 continuity + 4 drain + 3 determinism/metrics). Determinism verified; no N-gating needed (Boundary 1
is a pure function, Boundary 2 a seeded burst at 2000 subscribers).

## What was built

### Boundary 1 — `revision-continuity` × commit rate `cps` (pure function)
A reconnecting subscriber must replay from inside the **layered** recovery window `W + W_checkpoint`
(reactivity.md §Parent checkpoint summaries: the ring and the parent checkpoint *stack*, total
single-round-trip range = `256 + 4096 = 4352` revisions). A reconnect gap of `RECONNECT_GAP_SEC`
(= 60 s, the doc's recovery floor) at `cps` commits/sec is a lag of `round(cps · gap)` revisions;
`holds(cps)` is "`classifyResume` is not `OutOfWindow`" — the layered edge. Monotone-in-harm because
`round(cps·gap)` is non-decreasing in `cps`.

| axis | claim | edge `cps*` | design pt | margin |
|---|---|---|---|---|
| `commitRateCps` ∈ (0, 1000] | `revision-continuity` | **≈ 72.52** | 10 (nominal) | **+62.52** |

- **Layered vs replay-only bound (the headline nuance).** The boundary uses the authoritative layered
  bound. The far more conservative **replay-only** edge (`W / gap = 256/60 ≈ 4.27 cps`) is reported
  alongside as `replayOnlyEdgeCps`, and it sits *below* the nominal 10 cps — i.e. at nominal the
  replay ring alone (Backfill) already fails the 60 s gap (`replayCoverageSeconds = 25.6 s < 60`),
  while the stacked window covers it (`layeredCoverageSeconds = 435 s`, resume = `CheckpointWindow`).
  This *is* the adaptive-`W` finding (reactivity.md: fixed `W` drops below the floor at ≥ 10 cps). The
  spec pins `replayOnly < nominal < layered cps*`. Which bound the edge assumes is therefore explicit.
- At the located edge the layered window covers ≈ the gap (`layeredCoverageAtEdge = 60.01 s`); one
  step past the edge the resume is `OutOfWindow` (`continuityKindPastEdge`).
- Floor guarded strictly above 0 (silent topic ⇒ unbounded coverage; `coverageSeconds`/`evalContinuity`
  throw on `cps ≤ 0`, and the axis `lo = 1`). Reconnect-gap is a **named constant** (`RECONNECT_GAP_SEC`).

### Boundary 2 — `completes-within-drain` × `T_rejoin_jitter / T_drain` ratio (seeded burst)
On tail rotation the whole subscriber set re-registers at the new tail jittered over `T_rejoin_jitter`
while the old tail forwards until `T_drain`. `holds(ratio)` fixes `T_drain` and maps `ratio` to
`T_rejoin_jitter = round(ratio · T_drain)`, runs the seeded `simulateRotationBurst`, and returns its
`completedWithinDrain` (last arrival ≤ `T_drain`). The seed is threaded so `(seed, ratio)` is
byte-reproducible. Monotone-in-harm because `nextInt(b) = ⌊nextFloat()·b⌋` reuses the same per-draw
float, so the wave's last-arrival time is non-decreasing in `T_rejoin_jitter`.

| axis | claim | edge `ratio*` | design pt | margin |
|---|---|---|---|---|
| `rejoinDrainRatio` ∈ [0.1, 2.0] | `completes-within-drain` | **≈ 1.0000** | 0.5 (shipped 30 s/60 s) | **+0.50** |

- **De-tautologizing (the whole point — read `reactivity.ts` diff).** The shipped ratio cannot fail:
  arrivals land in `[0, T_rejoin_jitter)`, always before `T_drain` when jitter < drain. **And** the
  pre-existing `simulateRotationBurst` capped its run at `world.scheduler.run(T_drain)`, so arrivals
  scheduled past `T_drain` never fired and `completedWithinDrain` was *doubly* unable to be false. The
  fix changes the run horizon to `max(T_drain, T_rejoin_jitter)` so the genuine last-arrival time is
  observed when the wave spreads past drain. **For the shipped config (jitter 30 s < drain 60 s) the
  horizon is exactly `T_drain`, so the default burst — and the existing `reactivity.spec.ts` /
  `scenarios.spec.ts` assertions — are byte-for-byte unchanged** (verified: both still green).
- **Positive control flips it:** `holds(2.0) === false`; one step past the located edge the last
  arrival exceeds `T_drain` (`lastArrivalPastDrainEdge = 60105 ms > 60000`).
- **Drains via promotion fan-out, not a trivially-short window** (`drainViaFanoutAtDesign`): at the
  shipped ratio the new root fills to `cap_promote_fast` (32) and the tree spreads (`finalDepth > 0`).
  This is the non-vacuity witness (the `heal-convergence` structural-trap analogue).
- Slope lookahead stays disabled (inherited from `simulateRotationBurst` — not re-enabled), so the
  peak-root fast-promote mechanics the claim depends on are intact.

`runReactivityBoundaries(opts)` folds both into a `Metrics` sink via `recordBoundary` (keyed by
`(claim, axis)`) and returns the edge diagnostics. New surface exported from `index.ts`
(`runReactivityBoundaries`, `revisionContinuityAxis`/`tailDrainAxis`, `measureContinuityCoverage`/
`measureRotationDrain`, `replayOnlyEdgeCps`, `NOMINAL_CPS`, `RECONNECT_GAP_SEC`,
`SHIPPED_REJOIN_DRAIN_RATIO`, and the option/readout types).

## Use cases for testing / validation / usage

- **Each boundary finds a finite edge with the expected margin sign** — `boundaryFound`, margin > 0,
  `designInsideEnvelope`, `monotoneViolated === false`, edge in the expected band (cps ≈ 72.5;
  ratio ≈ 1.0).
- **Positive controls (not vacuous)** — continuity: `holds(10)` true / `holds(1000)` false, and 100
  cps ⇒ `OutOfWindow`. Drain: `holds(0.5)` true / `holds(2.0)` false; the past-edge probe outlasts
  `T_drain`.
- **Layered-vs-replay-only** — `replayOnly (4.27) < nominal (10) < layered cps* (72.5)`; replay-only
  coverage at nominal is already below the 60 s gap while layered is not.
- **Determinism** — `runReactivityBoundaries` byte-identical across two runs (boundaries + all
  diagnostics); `measureRotationDrain` identical for a fixed `(seed, ratio)`, and varies across seeds.
- **Metrics folding** — both boundaries present in the sink under their `(claim, axis)` tags.
- **Usage**: `runReactivityBoundaries({ seed })` for the default report; the two `*Axis` builders to
  drive `findBoundary` directly; `measureContinuityCoverage(cps)` / `measureRotationDrain(ratio)` for
  raw per-value diagnostics; `replayOnlyEdgeCps()` for the conservative bound.

## Known gaps / honest caveats (reviewer's starting points, not a finish line)

- **`simulateRotationBurst` was modified (not just reused).** The run-horizon change is small and the
  default path is provably unchanged, but it *is* a behavior change to a shared model function under a
  ticket that nominally said "new measurement, not new modeling." It was necessary — without it the
  drain claim is unfalsifiable. A reviewer should confirm the horizon change has no effect on any
  other caller (only `TailRotationScenario` and the burst's own spec call it; both use the default
  config and stay green) and decide whether it belongs here or in
  `simulator-strengthen-scenario-adversariality` (the cross-ref ticket that owns drain-claim
  hardening). If that ticket also touches the burst, watch for a merge seam.
- **Boundary 1 reuses the claim *name* `revision-continuity` for a different facet.** The
  `TailRotationScenario` claim of that name checks gap-free *monotone delivery*; this boundary bounds
  the *reconnect-gap coverage* facet (the cps-sensitive one). Same claim label, distinct mechanism —
  intentional (it's the parent table's "reactivity `revision-continuity`" row) but a reader could
  conflate them. Confirm the naming is acceptable or split the label.
- **`holds` uses the classifier (`< W + W_checkpoint`) not `coverageSeconds ≥ gap`.** They agree to
  within one revision / ~0.02 cps at the edge (the ticket said "and/or"); the classifier is the
  authoritative layered path. The `round(cps·gap)` lag introduces a ½-revision rounding at the very
  edge — `cps*` is pinned to ≈ 72.52 (tolerance 0.01), don't over-read the last digit.
- **Drain edge is RNG/population-sensitive.** `ratio* ≈ 1.0001` because with 2000 subscribers the max
  jittered arrival is ≈ 0.9998 · `T_rejoin_jitter` (matching the doc's 29,995 ms worked example).
  Fewer subscribers ⇒ a slightly larger `ratio*` (the wave's tail is thinner); the spec asserts a band
  (0.9–1.1), not a point. The edge is deterministic per `(seed, subscriberCount)` but shifts with
  either.
- **Drain `holds` is the arrival-window comparison, run through the real burst.** It is genuine drain
  *dynamics* (seeded jitter, fast-promote fan-out) rather than an analytic `jitter vs drain` compare,
  and the fan-out witness proves the mechanism engaged — but the binding quantity that flips the claim
  is still "did the last arrival beat `T_drain`." A reviewer who wants the claim to also depend on,
  e.g., per-arrival absorption latency or queue depth at the new tail would file that as a follow-up
  (it would lower `ratio*`).
- **`probePastEdge` has an unused-in-practice `integer` branch.** Both axes are real-valued, so it is
  always called with `integer=false` (an earlier draft passed `true` for drain and probed the ceiling
  — fixed; `lastArrivalPastDrainEdge` is now 60,105 ms, just past the edge, not 119,980 ms at ratio 2).
  The integer branch is dead for these two axes; kept only to mirror the harness convention.
- **`boundary.ts` reused unchanged** — this ticket added no harness surface, only two axis specs. The
  `simulator-envelope-core` cross-ticket `boundary.ts` ownership note (see the tree review) still
  applies upstream; nothing here depends on core's R\* axis.
