## What was built

A single spec file, `packages/db-p2p/test/empty-state-contract.spec.ts`, with one `describe` block per layer pair and two `it`s per pair (unknown-block + pending-only-metadata). A ninth test (`Tracker.tryGet`) rides on the last pair. All nine are small, self-contained, and have explicit 3s per-test timeouts under a 5s suite default.

### Layer pairs covered

1. **BlockStorage ↔ StorageRepo** — direct `MemoryRawStorage` + `BlockStorage` + `StorageRepo`, no mesh.
2. **StorageRepo ↔ CoordinatorRepo** — 1-node mesh via `createMesh(1, ...)`, observed through `mesh.nodes[0].coordinatorRepo.get`.
3. **CoordinatorRepo ↔ NetworkTransactor** — 1-node mesh + `buildNetworkTransactor` helper, observed through `transactor.get`.
4. **NetworkTransactor ↔ Collection** — `TransactorSource.tryGet` over the network transactor; `Tracker.tryGet` layered over the source with empty transforms.

### Contracts pinned (current behavior; documented inline above each assertion)

- **Unknown block**: every `.get(...)` layer returns `{ [id]: { state: {} } }`. No throw, no hang. `TransactorSource.tryGet` and `Tracker.tryGet` collapse this to `undefined`.
- **Pending-only metadata** (metadata seeded by `savePendingTransaction`, no commit): indistinguishable from "does not exist" at the no-context read path per the ticket-5 fix. Every layer yields `{ state: {} }`; `state.pendings` is NOT populated unless a committed revision exists. Collection-level `tryGet` returns `undefined`.
- On a 1-node mesh, the `CoordinatorRepo.fetchBlockFromCluster` fallback terminates cleanly for unknown/pending-only blocks — no infinite self-query loop.
- `NetworkTransactor.get` merge path (`network-transactor.ts:118-149`) treats `{ state: {} }` as a valid complete response: the block id appears in `resultEntries`, `missingIds` is empty, no aggregate error thrown.

### No latent contract mismatches surfaced

Every layer pair already honors the pinned contract on `main`. No `xit` / deferred-fix tickets were needed — the ticket-5 fix closed the last known gap. The spec now serves as a forcing function so future refactors can't reintroduce drift.

## Usage

- Pure test file; no production change.
- Future refactors that alter any layer's empty-state representation (e.g., swapping `{ state: {} }` for `undefined`, or adding `state.pendings` on no-context reads) will fail here first — at the contract seam, not in a downstream canary.
- To extend for a new layer (e.g., a caching layer inserted between `CoordinatorRepo` and `StorageRepo`), add a fifth `describe` block with the same two-`it` shape.
