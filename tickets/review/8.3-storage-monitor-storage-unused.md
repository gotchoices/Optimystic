## What was built

`StorageMonitor.estimateUsedSpace()` (`packages/db-p2p/src/storage/storage-monitor.ts:50`) was a stub returning 0; it now calls `this.storage.getApproximateBytesUsed?.() ?? 0`. The storage field — previously `_storage` (lint-silenced unused) — has been restored to `private readonly storage`.

`IRawStorage` (`packages/db-p2p/src/storage/i-raw-storage.ts`) gained an optional `getApproximateBytesUsed?(): Promise<number>` method. Optional preserves source compatibility with any out-of-tree implementer; `StorageMonitor` defends with `?.()` and `?? 0`.

`getCapacity()` override gating was also tightened. Previously, supplying just `totalBytes` (a disk-size hint) put the monitor entirely into override mode and suppressed any backend lookup — used always reported 0. Now used/available overrides are honored independently of `totalBytes`, so `totalBytes` alone lets you cap the disk size while still reading actual used bytes from the backend. This is the production-meaningful path that was previously broken.

### Backend implementations

- **FileRawStorage** (`packages/db-p2p-storage-fs/src/file-storage.ts`): recursive `fs.readdir(..., {withFileTypes:true})` + `fs.stat(file)` over `basePath`, summing file sizes. ENOENT swallowed (during concurrent deletion or on a fresh node before basePath exists). O(n) per call — caller (`StorageMonitor`) is invoked only at ring-transition checkpoints, so cost is amortized.
- **MemoryRawStorage** (`packages/db-p2p/src/storage/memory-storage.ts`): sums key lengths plus `JSON.stringify(value).length` across each of its five maps. This is an approximation of heap footprint — not byte-exact, but the call site only needs a usable proxy for capacity ratios.
- **MMKVRawStorage** (`packages/db-p2p-storage-rn/src/mmkv-storage.ts`): iterates `mmkv.getAllKeys()`, filtering by `this.prefix`, summing key + stored-string length.

### libp2p-node-base wiring (unchanged)

`libp2p-node-base.ts:386` already passed the live `rawStorage` to `new StorageMonitor(...)`. No call-site change needed.

## Tests

`packages/db-p2p/test/storage-monitor.spec.ts` (new) — six tests:
1. Reflects backend-reported bytes when no overrides are supplied.
2. Treats missing `getApproximateBytesUsed` (older backends) as 0 — `available === total`.
3. Clamps reported used to `total` when backend overshoots (defensive).
4. Floors negative reports at 0.
5. `usedBytes` override wins; backend is **not** called.
6. `availableBytes` override yields `used = total - available`.

## Use cases / scenarios

1. **Real production path** (post-fix): node starts, FileRawStorage reports actual on-disk size, StorageMonitor returns it as `used`, RingSelector's `shouldTransition()` can finally trigger `direction: 'out'` when `used/total > 0.85`.
2. **Capacity hint without backend disable**: operator configures `totalBytes: 50GB` (a disk-size cap). Backend is still consulted for used. Previously this combination forced `used=0`.
3. **Custom override** (testing or operator-known capacity): explicit `usedBytes` short-circuits the backend — useful for test fixtures and dry-run sizing.
4. **Backend without implementation** (hypothetical out-of-tree IRawStorage): `getApproximateBytesUsed?.()` returns undefined → `?? 0` → behavior degrades to pre-fix (no transitions), but doesn't crash. A future enhancement could log a one-shot warning.
