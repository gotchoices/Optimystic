## What was done

Pure documentation ticket — no code changes. Three files edited:

### `docs/transactions.md`

Added a new `## Read Consistency and Staleness` section between "Client Synchronization" and "Key Components". Covers:

- **Non-responsible reads (soft serves):** node serves blocks even when not the DHT-responsible node; no caller error, only internal `proximity:get-warning` log.
- **Lazy read-repair window:** default `lazy` mode trusts local copy for `readRepairWindowMs` (default 10 000 ms) before re-verifying with the cluster.
- **Worst-case staleness bound:** up to one commit behind within the window.
- **Configuration table:** `readRepairMode`, `readRepairWindowMs`, `readRepairSampleRate` — with `paranoid` as the no-staleness option.
- **Transaction protection vs. bare reads:** explains that transaction read-dependencies are validated at commit (catches stale data), but bare reads carry no such check. Points to `correctness.md` Theorem 15 (Read-Path Integrity).

### `packages/quereus-plugin-optimystic/README.md`

Added a blockquote callout under the Transactions section cross-linking to the new `docs/transactions.md` section. States the 10 s default window, distinguishes transaction vs. bare reads, names the `paranoid` knob.

### `docs/right-is-right.md`

Added one bullet to the "Open Design Questions" list pointing gossip/reputation-propagation design to `tickets/backlog/speculative/7.5-gossip-reputation-blacklisting.md`. Does not restate or re-argue the design.
