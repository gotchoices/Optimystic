## What was built

### The harness (`boundary.ts`) ‚Äî the shared foundation
`findBoundary(spec)` drives one monotone-in-harm stress axis: a geometric (or small-integer linear)
bracket scan to find the first pass‚Üífail transition, then bisection to the **last value that still
holds** (`criticalValue`). Reports `margin = criticalValue ‚àí designAssumption`, `marginRatio` (NaN
when designAssumption 0), `designInsideEnvelope`, `boundaryFound`, `monotoneViolated`, `evaluations`.
Three open-bracket cases are explicit (not mis-reported): fails-at-`lo` ‚Üí `criticalValue < lo`,
negative margin; holds-to-`hi` ‚Üí `boundaryFound=false`, `criticalValue=hi` (a lower bound);
non-monotone (a pass past an observed fail) ‚Üí `monotoneViolated=true`, first-fail edge still
returned. `recordBoundary` folds a readout into a `Metrics` sink keyed by `(claim, axis)`.
`decreasing-harm` is supported via internal reflection (rarely-used branch, lightly tested).

### Three tree axes (`boundary-tree.ts`)
| Axis | claim | edge found @ N=2000 | design pt | margin sign |
|---|---|---|---|---|
| `prefixSkew` ‚àà[0,1] | `depth-law` | ‚âà 0.042 | 0 (uniform sha256) | + |
| `churnRate` ‚àà[0,1] | `promotion-demotion-stable` | ‚âà 0.499 | 0 (settled tree) | + |
| `unwillingFraction` ‚àà[0,1] | `no-give-ups` ‚àß `‚â§d_max+2` hops | ‚âà 0.48 | 0 (all willing) | + |

- **prefix-skew** ‚Äî `skewedLadder(index, dMax, F, skew, seed)` (new, beside `uniformLadder`): at
  skew `s`, each participant uses the hot index with probability `s` (seeded coin) else its own ‚Äî
  switching the *whole effective index*, so the result is literally a `uniformLadder` (valid nesting
  at every skew, incl. `s=1`; no throw/corruption). `holds(s)` grows a full tree via `register` and
  checks `maxOccupiedTier === expectedDepth(N,F,cap)`.
- **churn** ‚Äî a two-tier single branch with a tier-1 cohort held at `cap_promote`; each cycle drains
  `‚åär¬∑pop‚åã`, ages the clock past a **shortened** `T_demote` (`tDemoteMs=gossipRoundMs`,
  `tPromoteStickyMs=0`, so the hysteresis is the thing under test), then refills. `holds(r)` =
  `PromotionTracer.result().oscillations === 0`. `measureChurnFlap` exposes `{oscillations,
  demotions}` ‚Äî the demotion count is the non-vacuity witness.
- **unwilling** ‚Äî grows a tree, runs `walkSamples` (100) `ParticipantWalk`s with a `WalkAdmission`
  oracle that replies unwilling for a deterministic fraction `f` of `(coord, memberAttempt)` members.
  `holds(f)` = `gaveUp===0 && maxHops ‚â§ d_max+2`. `runTreeBoundaries.unwillingBreach` records which
  sub-condition flips first; `measureUnwillingWalks` exposes the readout.

`runTreeBoundaries(opts)` folds all three into a `Metrics` sink and **N-gates the two full-tree
axes** (`prefixSkew`, `unwillingFraction`) by `treeSampleMaxN` (default 10k), recording the skip
(`boundary.skipped` counter + `skipped[]`); the single-cohort churn axis is never gated. Exports
added to `index.ts`.

## Use cases for testing / validation / usage

- **Each boundary finds a finite edge with the expected margin sign** ‚Äî covered for all three
  (`boundaryFound`, `criticalValue ‚àà (0,1)`, `margin > 0`, `monotoneViolated === false`).
- **Positive controls (not vacuously stable)** ‚Äî uniform skew matches the law / skew 0.5 deepens it;
  churn 0 flat / churn 0.8 flaps **and** demotes (proves demotion reachable); f=0 succeeds / f=1.0
  ‚Üí all walks give up and blow the hop bound.
- **Determinism** ‚Äî `runTreeBoundaries` is byte-identical across two runs; harness deterministic on
  pure predicates.
- **N-gating** ‚Äî a 100k run skips the full-tree axes and records the skip; churn still runs.
- **Harness edge cases** ‚Äî exact edge (last-pass), fails-at-lo (negative margin), held-to-hi (lower
  bound), non-monotone detection, integer-adjacent + real-tolerance + tolerance‚â•range termination,
  inverted-range throw, decreasing-harm smoke, `recordBoundary` fold.
- **Usage**: `runTreeBoundaries({ N, seed })` for the default report; `prefixSkewAxis` /
  `churnFlapAxis` / `unwillingFractionAxis` to plug into `findBoundary` directly;
  `measureChurnFlap` / `measureUnwillingWalks` for raw per-value diagnostics.

## Known gaps / honest caveats (the reviewer's starting points, not a finish line)

- **Churn axis measures *depth* oscillation, not promote/demote-event flapping.** Per the ticket it
  reuses `PromotionTracer.oscillations` (observed-depth decreases). A cohort can promote‚Üídemote
  without depth changing: for `r ‚àà [~0.375, 0.5)` real demotions occur (count drops ‚â§ `cap_demote`)
  yet `holds` stays true because the deepest cohort never fully drains to 0 participants. The edge
  `r*‚âà0.5` is where the deep cohort hits **0** (depth 1‚Üí0). If the reviewer thinks the claim should
  trip on promote/demote *events* (not depth), that's a spec re-interpretation worth a fix ticket ‚Äî
  it would lower `r*`. The non-vacuity control proves demotion is genuinely exercised regardless.
- **Churn boundary is effectively a step** at `pop/q` granularity (counts are integers), so `r*` is
  pinned to ‚âà0.5 ¬± tolerance rather than a smooth curve. Fine for `findBoundary` (monotone step), but
  don't over-read its precision.
- **Unwilling: the hop bound breaches before walks give up** (`unwillingBreach === 'hop-bound'` at
  these defaults). Each `unwilling_member` retry costs a hop, and a grown N=2000 tree's base walk
  uses ~3 hops against a bound of `d_max+2=6`, so ~3 consecutive unwilling members at one coord blow
  it. The edge `f*‚âà0.48` therefore reflects the *hop-bound* fragility, not give-ups (those need
  `f‚âà1`). The value is **deterministic but sensitive to `walkSamples` and the coord-hash
  distribution** ‚Äî a different sample count shifts `f*`. Consider whether the hop-bound belongs in
  the same predicate as give-ups or should be its own axis.
- **Skew model is one specific non-uniformity** (a single hot leaf via whole-index switch), not a
  general prefix distribution. It is monotone and valid-nesting, but a reviewer may want a
  distribution-shaped skew (e.g. Zipf over buckets) as a follow-up; `s*` would change.
- **`decreasing-harm`** has one smoke test only (no subsystem uses it; all four axes are
  increasing-harm).
- **`boundary.ts` / `boundary.spec.ts` ownership** ‚Äî see the cross-ticket note. The harness's own
  R\* reference axis is *not* here (it's core's). If core never lands, this harness is still
  self-validated by `boundary.spec.ts`.
- **No 100k+ opt-in run was executed** (gated by default and slow); only the *skip path* is tested.
  A human/CI can run `runTreeBoundaries({ N: 100000, treeSampleMaxN: 100000 })` out-of-band to
  confirm the large-N edges, but it is not agent-runnable inside the idle-timeout budget.
