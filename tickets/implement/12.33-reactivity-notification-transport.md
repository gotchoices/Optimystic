# Reactivity notification transport — live node wiring

## Goal

Compose `reactivity-notify-protocol` + `reactivity-forwarder-host` + `reactivity-pushstate-gossip` into
the production libp2p node so reactivity origination, fan-out, inbound delivery, and PushState gossip are
**live** for every collection on a `cohortTopic`-enabled node. This is the companion to
`cohort-topic-host-node-wiring`: that bridge fires `onLocalCommit`; this ticket is what the emitted
notifications actually travel over. It also pins the load-bearing subscriber-side topic-bytes encoding
and un-skips the real-libp2p socket-delivery test.

This ticket keeps the slug `reactivity-notification-transport` so the downstream
`reactivity-backfill-resume-rpc-transport` (12.4) and `reactivity-tail-rotation-transport` (12.5)
prereqs resolve unchanged.

## Wiring (in `libp2p-node-base.ts`, the `if (cohortEnabled)` block, ~L714-779)

After `attachCohortChangeBridge` (which already installs the membership gate + cert extractor), and using
the exposed `host` (`host.service`, `host.registry`, the cohort gossip transport):

1. **Notify transport.** `const notify = new Libp2pReactivityNotifyTransport(node, { selfPeerId })`.
2. **Forwarder host.** `const fwd = new ReactivityForwarderHost({ transport: notify, selfPeerId, profile,
   pushStateInit, verifierFor, directSubscribers, resolveChildPrimary, deliverLocal })`:
   - `profile` — the node's `NodeProfile` (Edge ⇒ subscriber-only via the policy gate).
   - `pushStateInit(topicId, n)` — `{ collectionId: n.collectionId, topicId: bytesToB64url(topicId),
     tailIdAtJoin: n.tailId, deltaMaxBytes: reactivityNodePolicy(profile).deltaMaxBytes }`.
   - `verifierFor(topicId)` — `createNotificationVerifier({ verifier: host.service.verifier(), tier: T3 })`.
   - `directSubscribers(topicId)` — `host.registry.findServing(topicId, T3)?.records(topicId)` (or the
     coord-resolving variant) filtered to reactivity `appState` → **`bytesToPeerIdString(participantId)`**
     (peer-id string, NOT `bytesToB64url` — see the dial-target encoding constraint below).
   - `resolveChildPrimary(ref)` — `ref.primary ?? <FRET cohort around b64urlToBytes(ref.coord), pick
     primary>` (reuse the cohort gossip transport's `CohortPeerResolver`/FRET assembly), returned as a
     **peer-id string**. No-op until the parent/child link lands; wire the call anyway.
   - `deliverLocal(topicId, n)` — route to the node's subscriber registry (step 4).
3. **Origination emit.** Construct `ReactivityOriginationManager`:
   - `resolveContext(event)` — derive `tailId = reactivityTailBytes(event.tailId)` (**MUST** reuse the
     gate's `reactivityTailBytes`, NOT db-core `blockIdToBytes` — see load-bearing constraint below),
     `deltaMaxBytes` from `reactivityNodePolicy(profile)`; `rotationHint` is left to 12.5.
   - `emit: (n) => void fwd.ingest(reactivityTopicId(reactivityTailBytes(<tailId from n>)), n)` — bind to
     the forwarder host's `ingest`. `.install()` (sets `service.onLocalCommit`).
4. **Subscriber registry + inbound routing.** Add a node-level `ReactivitySubscriberRegistry`:
   `register(topicId, onNotification)` / `unregister`, keyed by topicId-b64. The notify inbound handler
   (`registerNotifyHandler(node, protocols.notify, notify)`) plus `notify.onNotification((from, n) =>
   fwd.onInbound(from, n))`, and `fwd`'s `deliverLocal` looks the manager up in this registry and calls
   `manager.onNotification(n)`. (The Quereus `Database.watch` → manager *bridge* stays the backlog item
   `optimystic-network-reactive-watch-integration-test`; this ticket provides the registry + socket
   routing so a constructed manager receives socket-delivered notifications.)
5. **PushState gossip.** Construct `ReactivityPushStateGossipDriver` over the host's cohort gossip
   transport (`broadcastOver`), `liveCollections`/`pushStateForGossip` from `fwd`'s PushState map, and an
   `isCohortMember` gate from FRET assembly; `registerNotifyHandler`-style over `protocols.pushStateGossip`
   → `driver.deliver`; `driver.start()`.
6. **Register + teardown.** `node.handle` the reactivity protocols; extend the existing `node.stop`
   wrapper to `driver.stop()` + `node.unhandle(reactivityProtocolList(protocols))` **before**
   `host.stop()` and the previous stop (timers/handlers released before transports close).

## Load-bearing constraint: subscriber-derived topic bytes MUST match origination

Origination derives the reactivity coord from `reactivityTailBytes(tailId) = new TextEncoder().encode(tailId)`
(`cohort-topic/reactivity-membership-gate.ts:46`), **not** db-core's async `blockIdToBytes` (which
`sha256`s first → a double hash). The subscriber side MUST feed `reactivityTopicId` the SAME bytes when
deriving the cohort it subscribes to from a `BlockId` `tailIdAtAttach`. `ReactivitySubscriptionManager`
already takes raw `tailIdAtAttach: Uint8Array` and applies `reactivityTopicId` to it — so the contract is
on the **production subscribe factory** that converts a `BlockId` to those bytes: it MUST call
`reactivityTailBytes(tailId)`. Export `reactivityTailBytes` from the reactivity index (or a shared
`reactivity/topic-bytes.ts`) so the subscribe path and the gate share one function, and document the
contract on `ReactivitySubscriptionManager`'s `tailIdAtAttach` JSDoc.

If origination and subscription derive `coord_0(H(tailId ‖ "reactivity"))` from differently-encoded tail
bytes, they resolve different coords and **origination silently never reaches subscribers** — green tests,
dead feature. The asserting test (below) is mandatory.

## Load-bearing constraint: subscriber-id / dial-target encoding MUST match the transport

`ReactivityForwarderHost` is **encoding-agnostic** — `selfPeerId`, the `directSubscribers(topicId)` output,
the `resolveChildPrimary(ref)` result, and every `transport.send(target, …)` target are one opaque string
space, but the host does not pick which. The **transport** does:
`Libp2pReactivityNotifyTransport.send` calls `peerIdFromString(target)` (and the self-guard compares
`target === selfPeerId`). So all four seams MUST be the **canonical peer-id string** space.

The trap: cohort member-id bytes are `peerIdToBytes(peerId) = utf8(peerIdString)` (`cohort-topic/peer-codec.ts`),
and the convenience adapter `reactivityDirectSubscribers` returns **`bytesToB64url(participantId)`** =
base64url(utf8(peerIdString)) — **not** a peer-id string. Feeding that to `peerIdFromString` throws, the
transport swallows it (`log("dropped a send …")`), and **origination silently never reaches subscribers**
(green tests, dead feature — exactly the tail-bytes failure mode, one layer down).

Pick the peer-id-string space end-to-end (do **not** use the base64url adapter output raw):
- `directSubscribers(topicId)` — filter the cohort records to reactivity `appState` and map with
  **`bytesToPeerIdString(record.participantId)`** (from `cohort-topic/peer-codec.ts`), NOT
  `reactivityDirectSubscribers` / `bytesToB64url`. (If you keep `reactivityDirectSubscribers`, it must be
  changed to emit peer-id strings, and its spec updated.)
- `selfPeerId` — the node's peer-id **string** (`node.peerId.toString()`), matching the transport's own
  `selfPeerId` self-guard.
- `resolveChildPrimary(ref)` — return a peer-id string (decode `ref.primary` via `bytesToPeerIdString` if it
  is carried as member bytes).
- **Mandatory assertion** (add to the node-wiring or encoding spec): a `directSubscribers` entry for a known
  member round-trips through `peerIdFromString(target)` without throwing — i.e. assert the target IS a valid
  peer-id string, pinning the no-dial regression both ways (valid peer-id string passes; a base64url-of-bytes
  target throws). This is the analogue of the tail-bytes coord-equality test for the *dial-target* space.

## Mock-tier vs. real-libp2p (resolved)

- **Mock-tier harness keeps modeling the transport in-process.** The cohort-topic mesh is an in-process
  FRET mesh (no real sockets — `cohort-topic-mesh-harness.ts` routes activity in-process,
  `crashNode` just rejects dials), so the notify transport (real libp2p dials) cannot run there. The
  `reactivity-e2e-mock-tier` suites already drive the real hot path and *model* exactly this transport;
  leave that approach unchanged (it is the **mock** tier by definition). No harness rewrite.
- **Real sockets: un-skip the delivery test.** `substrate-real-libp2p.integration.spec.ts` (env-gated,
  real TCP) currently `it.skip`s notification socket delivery because "production registers no emit
  transport / subscriber-delivery protocol." That is now false. Un-skip / add: a commit on a real tail
  cohort member fires a `NotificationV1` that reaches a remote subscriber over a real socket (subscriber
  constructed against the remote node, registered in the subscriber registry, receives + verifies). The
  broader at-scale real-libp2p regime stays `substrate-e2e-real-libp2p-tier`'s.

## Edge cases & interactions

- **`cohortTopic` disabled (default):** no reactivity transport, no handlers, no timer — unchanged node.
- **Edge node:** subscriber-only (policy gate → no PushState, never forwards); still receives + delivers
  over the notify protocol. deltaMax = 0.
- **`onLocalCommit` single-owner:** the origination manager `.install()` overwrites the hook; ensure no
  other consumer races for it on the same node.
- **Non-member commit / tail-less event:** the change-bridge gate already returns before `emit` (no
  fan-out).
- **Teardown ordering:** reactivity timers + protocol unhandle BEFORE `host.stop()` BEFORE the previous
  stop wrapper (transports close last) — compose with the existing arachnode/clusterMember stop wrappers
  (each calls its captured previous stop last).
- **Subscriber co-located with origin:** `deliverLocal` (in-process), never dialed (selfPeerId guard).
- **A subscriber registered for a topic this node also forwards:** both roles run (idempotent via the two
  dedupe layers).
- **Double-hash regression:** the coord-equality test is the guard; also assert origination's gate and the
  subscribe factory call the *same* `reactivityTailBytes`.

## Honest gaps to flag for review

- Child-cohort fan-out path is wired but exercised only synthetically until `cohort-topic-parent-child-link`
  populates `PushState.childCohorts` (single-tier-0 reach).
- **Inbound forwarder-role gating:** step 4 wires `notify.onNotification((from, n) => fwd.onInbound(from, n))`
  for *every* inbound notify. `onInbound` runs the forwarder role unconditionally; the host self-gates only
  on the Edge policy and an empty `directSubscribers`. A Core node that receives a notify for a topic it does
  **not** serve will instantiate a `PushState` and fan out to an empty set (harmless + cached, but wasteful).
  Either route inbound notifies to `onInbound` only for served topics, or accept the no-op. Confirm the
  intended behavior.
- **Backpressure model:** `ReactivityForwarderHost.fanOut` enqueues onto each per-subscriber `BoundedQueue`
  then **drains it inline** and fires `transport.send` fire-and-forget — so in steady state the queue is a
  pass-through and drop-oldest is not organically reachable (it only triggers if a queue is already full).
  This matches the forwarder-host ticket's pseudo-code, but if genuine in-flight backpressure is intended
  (bounded concurrent dials per subscriber, the queue absorbing a slow dial), that is a follow-on redesign
  — a per-subscriber async delivery worker. Decide which model is intended before relying on the queue for
  real backpressure under load.
- PushState-gossip frame peer-sig envelope signing is deferred (membership gate + self-verifying entries;
  hardening backlog ticket filed by `reactivity-pushstate-gossip`).
- The Quereus `Database.watch` → `ReactivitySubscriptionManager` production bridge remains the backlog item
  `optimystic-network-reactive-watch-integration-test`; this ticket delivers only the node-level subscriber
  registry + socket routing it will plug into.
