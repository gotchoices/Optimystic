# Reactivity forwarder host (receive → forward → fan-out orchestration)

## Goal

Build the **orchestrator** that turns the in-process forward *decision* into live fan-out: take a
`NotificationV1` (from local origination or an inbound dial), run the db-core forwarder receive path
(verify → dedupe → buffer), and on a `"forward"` decision dial every direct subscriber and every child
cohort over the `ReactivityNotifyTransport` from `reactivity-notify-protocol`. It owns the
per-subscriber dequeue→dial delivery loop, the per-subscriber bounded-queue backpressure interaction,
subscriber-queue eviction (the memory bound), the lazy `PushState` instantiation behind the Edge policy
gate, and the inbound role decision (forwarder-primary vs. direct-subscriber, possibly both).

This ticket does **not** touch the libp2p node assembly (that is `reactivity-notification-transport`) or
gossip (that is `reactivity-pushstate-gossip`). It depends only on the transport interface + db-core
logic, so it is unit-testable with a fake transport.

## Background (already implemented — reuse, do not rebuild)

- `createReactivityForwarder({ state, verifier }).receive(n, now): "forward" | "duplicate" | "untrusted"`
  (`forwarder.ts`) — verify → dedupe → append to the replay ring; the **decision** is done, only the
  `"forward"` transport binding is missing.
- `PushState` (`push-state.ts`): `replayBuffer`, `dedupe`, `childCohorts: CohortRef[]`,
  `enqueueForSubscribers(ids, n)`, `perSubscriberQueue` (`backpressure.ts`: `fanOut`, `peekQueue(id)`,
  `remove(id)`; `BoundedQueue.drain()` / `.dropped`).
- `instantiateForwarderPushState(profile, init): PushState | undefined` (`policy.ts`) — returns
  `undefined` on an Edge node (T3 producer forbidden); this is the gate for *becoming* a forwarder.
- `CoordEngine.records(topicId): RegistrationRecord[]` (`host.ts:258`) — the cohort's direct
  registration records; `record.appState` carries the opaque subscribe payload, decodable with
  `decodeSubscribeAppPayload` (db-core wire.ts) and filtered to `kind == "reactivity"`. `record.participantId`
  is the **subscriber's dialable peer id** (db-core service.ts pins `participantId = self`); that is the
  fan-out target and the `perSubscriberQueue` key (base64url).
- The reactivity mesh harness (`reactivity-mesh-harness.ts` `drainFanOut`/`drainSubscriber`) is the exact
  in-process model of the loop this ticket makes real: `forwarder.receive(n)` →
  `perSubscriberQueue.enqueue(subId, n)` → drain → `manager.onNotification(n)`. Match its semantics.

## What to build: `packages/db-p2p/src/reactivity/forwarder-host.ts` (new)

```
interface ReactivityForwarderHostDeps {
  transport: ReactivityNotifyTransport;
  selfPeerId: string;                              // base64url member id; never dialed, delivered in-process
  profile: NodeProfile;                            // gates PushState instantiation (Edge → subscriber-only)
  /** Build the per-collection PushState init when this node first ingests for a topic it serves. */
  pushStateInit: (topicId: Uint8Array, n: NotificationV1) => PushStateInit;
  /** Notification verifier for a collection (createNotificationVerifier over the host's service verifier, tier T3). */
  verifierFor: (topicId: Uint8Array) => NotificationVerifier;
  /** Direct-subscriber participant ids (base64url) for a topic — reactivity records from CoordEngine.records. */
  directSubscribers: (topicId: Uint8Array) => string[];
  /** Resolve a child cohort's primary peer id to dial (CohortRef.primary, else FRET cohort resolver). */
  resolveChildPrimary?: (ref: CohortRef) => string | undefined;
  /** Route an inbound notification to a co-located subscription manager, if this node subscribes the topic. */
  deliverLocal?: (topicId: Uint8Array, n: NotificationV1) => void;
  clock?: () => number;
}

class ReactivityForwarderHost {
  /** Local origination emit (ReactivityOriginationManager.emit binds here) AND inbound forwarder path. */
  ingest(topicId: Uint8Array, n: NotificationV1): Promise<void>;
  /** Inbound notify dial: forwarder role (ingest) and/or subscriber role (deliverLocal). */
  onInbound(from: PeerRef, n: NotificationV1): Promise<void>;
  /** The live PushState for a served topic, or undefined (Edge / not yet ingested). Test/diagnostic. */
  pushStateFor(topicId: Uint8Array): PushState | undefined;
}
```

### `ingest(topicId, n)` — the unified receive→forward→fan-out

1. **Lazy PushState + forwarder.** On first ingest for `topicId`, call
   `instantiateForwarderPushState(profile, pushStateInit(topicId, n))`. `undefined` (Edge) ⇒ this node
   never forwards; return immediately (subscriber-only nodes do delivery through `onInbound`'s
   `deliverLocal`, not here). Cache `{ pushState, forwarder }` per topic.
2. **Serialize per-topic.** Process ingests for one topic strictly in sequence (a tiny per-topic async
   queue / promise chain): `forwarder.receive` is async (verify), and the replay ring + dedupe set must
   not interleave across two concurrent notifications for the same collection.
3. `decision = await forwarder.receive(n, now)`. On `"duplicate"` or `"untrusted"` → stop (no fan-out, no
   buffer mutation beyond what `receive` already did/declined).
4. On `"forward"` → **fan out**: `fanOut(topicId, pushState, n)`.

### `fanOut(topicId, pushState, n)`

- Resolve `subscriberIds = directSubscribers(topicId)`.
- **Reconcile queues (memory bound):** drop any `perSubscriberQueue` entry whose id ∉ `subscriberIds`
  (`perSubscriberQueue.remove(id)`) — a departed/TTL-expired/withdrawn subscriber's queue is reclaimed
  lazily each round, so the map can never grow past the live subscriber set + the current round. This
  satisfies the `cohort_subscribers × queue_max × notification_size` bound the 12.5 memory note flags.
- `pushState.enqueueForSubscribers(subscriberIds, n)` (drop-oldest under pressure, per-subscriber).
- For each `subId` in `subscriberIds`: drain its bounded queue (`peekQueue(subId)?.drain()`) and for each
  dequeued notification: if `subId === selfPeerId` → `deliverLocal?.(topicId, m)`; else
  `transport.send(subId, m)` (already failure-isolated). One slow/dead subscriber's queue must not block
  the loop for the others (drain + send per subscriber, isolate per-target).
- For each `ref` of `pushState.childCohorts`: `target = ref.primary ?? resolveChildPrimary?.(ref)`; if
  resolved and `!== selfPeerId`, `transport.send(target, n)` (the unmodified frame — forwarders never
  re-sign).

### `onInbound(from, n)`

- Derive `topicId` from the notification (reactivity topic is `reactivityTopicId(b64urlToBytes(n.tailId))`;
  expose a small helper or take topicId via the notify routing layer — the node-wiring ticket routes by
  topic). Run **both** roles as applicable:
  - **forwarder role:** if this node serves the topic's cohort (has/may-instantiate a PushState) →
    `ingest(topicId, n)`.
  - **subscriber role:** `deliverLocal?.(topicId, n)` (routes to a co-located
    `ReactivitySubscriptionManager.onNotification`).
  A node that is both a cohort member and a subscriber does both (in-process delivery via `deliverLocal`,
  fan-out via `ingest`); dedupe in the forwarder + `(collectionId, revision)` dedupe in the subscriber
  keep this idempotent.

## Edge cases & interactions

- **Edge node (T3 producer forbidden):** `instantiateForwarderPushState` → `undefined`; never forwards,
  never instantiates a `PushState`; still receives + delivers as a pure subscriber via `deliverLocal`.
- **Origin re-emit / duplicate inbound:** `forwarder.receive` dedupes (`"duplicate"`) → no double fan-out.
- **Untrusted (bad threshold sig):** `"untrusted"` → dropped before any buffer/dedupe mutation; never
  fanned out (a compromised forwarder cannot forge).
- **Subscriber co-located with origin (self):** delivered in-process (`deliverLocal`), never dialed.
- **Slow subscriber:** its bounded queue drop-oldest; `dropped` counter bumps; the fast subscribers'
  drains/sends are unaffected (per-subscriber isolation — the load-bearing property of §Slow-subscriber
  backpressure). A per-target `transport.send` failure is isolated too.
- **Departed / TTL-expired / withdrawn subscriber:** `directSubscribers` no longer lists it → its queue is
  removed on the next `fanOut` (lazy GC). Document that the map shrinks at most one round late.
- **Concurrent ingest for the same collection:** per-topic serialization keeps the ring/dedupe consistent;
  ingests for *different* collections proceed concurrently.
- **Child cohorts empty (single-tier-0 milestone):** `childCohorts == []` until the cohort-topic
  parent/child link lands (`cohort-topic-parent-child-link`, backlog) — child fan-out is wired but a no-op
  today; cover it with a synthetic `childCohorts` entry test so the dial path is exercised.
- **Partial fan-out then primary failure:** out of scope to *complete* here (cohort recovery + backfill
  via gossip covers it — `reactivity-pushstate-gossip` + 12.4); but the loop must not crash on a
  mid-loop send failure (isolate each).
- **`ingest` must never throw into the commit path:** origination emit is called from the change-bridge
  (which already isolates throws), but the host should also defensively isolate so a fan-out fault can
  never surface as a commit failure.
