# Hoist the duplicated wire-validation primitives into one shared module

The identical structural-validation primitive set — `asObject`, `requireV1`, `reqString`,
`reqFiniteNumber`, `reqStringArray`, the base64url helpers, the fail-with-`CohortWireError` shim, etc. —
is copy-pasted across **three** modules and has already diverged. Consolidate to a single shared module
that all three consume. This is part (c) of the parent fix ticket; it runs after
`cohort-topic-wire-hardening` (part (b)) so the fixed-length hardening lands once, in the shared module,
rather than being duplicated and then merged.

All three modules live in `packages/db-core/src` (the parent ticket mislabeled them as `db-p2p` — they
are all under `db-core`).

## The three copies

- **`cohort-topic/wire/validate.ts`** — the richest set: `fail`, `asObject`, `requireV1`, `reqString`,
  `optString`, `reqFiniteNumber`, `optFiniteNumber`, `reqBool`, `optBool`, `reqStringArray`,
  `optStringArray`, `reqEnum`, `assignDefined`, `b64urlField`, `b64urlFixedLen`. Plus the domain
  validators (`validateRegisterV1`, …) and domain-specific helpers `tier` / `treeTier` (these stay
  local — they encode cohort-topic tier semantics, not generic primitives).
- **`matchmaking/wire.ts` (~170-266)** — a local block explicitly commented "mirror
  cohort-topic/wire/validate.ts; local to keep matchmaking DRY": `fail`, `asObject`, `requireV1`,
  `reqString`, `reqBool`, `optBool`, `reqFiniteNumber`, `optFiniteNumber`, `reqStringArray`,
  `b64urlField`, `reqIntInRange`. Also owns `DEFAULT_MAX_APP_PAYLOAD_BYTES` and utf8 coder constants.
- **`reactivity/wire-validate.ts`** — already a standalone exported primitives module (closest to a
  clean base): `failWire`, `asObject`, `requireV1`, `reqString`, `reqBoolean`, `reqFiniteNumber`,
  `reqIntInRange`, `b64urlField`, `reqStringArray`. It already imports `CohortWireError` +
  `b64urlToBytes` from `cohort-topic/wire`.

## Divergences the hoist must reconcile — the real hazard

The copies are not textually identical; picking one canonical signature and migrating every call site
is where bugs hide:

- **`reqIntInRange` has two incompatible signatures.** reactivity: `reqIntInRange(obj, key, what, min,
  max?)` (reads `obj[key]`). matchmaking: `reqIntInRange(value, key, what, min, max?)` (takes an
  already-extracted number). Pick one; update all call sites accordingly. (The `obj,key` form composes
  better with the other `req*` helpers; recommend that, but check every caller.)
- **`reqStringArray` has two signatures.** cohort-topic + matchmaking: `reqStringArray(obj, key, what)`.
  reactivity: `reqStringArray(value, key, what)`. Same reconciliation.
- **Naming:** `fail` (cohort-topic, matchmaking) vs `failWire` (reactivity); `reqBool` (cohort-topic,
  matchmaking) vs `reqBoolean` (reactivity). Pick one name each and export; update imports.

## Suggested shape

Create one neutral primitives module — recommend `packages/db-core/src/cohort-topic/wire/primitives.ts`
(cohort-topic is where `CohortWireError` and `b64urlToBytes` already live, so no new cross-package
coupling). It exports the full generic set **including `b64urlFixedLen`** (so the part-(b) fixed-length
checks live here and any future consumer inherits them). Then:

- `cohort-topic/wire/validate.ts` imports the primitives from it; keeps only the domain validators and
  the `tier` / `treeTier` domain helpers. Re-export `CohortWireError` from wherever it currently lives
  so existing importers of `validate.ts` are unaffected.
- `matchmaking/wire.ts` deletes its local ~170-266 block and imports from the shared module; keeps
  `DEFAULT_MAX_APP_PAYLOAD_BYTES` and its utf8 coders (those are matchmaking-specific, not primitives).
- `reactivity/wire-validate.ts` becomes a thin re-export of the shared module (or is deleted and its
  importers repointed — grep for `wire-validate` importers first: `reactivity/wire.ts`, `push-state.ts`,
  `backfill.ts`, `resume.ts` per its module doc).

Keep the behavior identical — this is a consolidation, not a semantics change. The only intended
behavior delta already landed in part (b).

## Expected behavior

- One shared primitives module; the three consumers import from it; no duplicated primitive set remains.
- Every existing test across cohort-topic, matchmaking, and reactivity wire subsystems stays green.
- A future hardening tweak to a primitive (like part (b)'s length check) now lands in exactly one place.
