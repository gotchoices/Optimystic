# Reactivity operating-envelope boundaries

Two boundaries for reactivity (parent: `simulator-validity-envelope`, candidate table rows
"reactivity `revision-continuity`" and "reactivity tail-rotation drain"), driven by the `findBoundary`
harness from `simulator-envelope-core`. Reuse `coverageSeconds`, `classifyResume`/`RollingCheckpoint`,
`CohortPushState`, `simulateRotationBurst` — new measurement, not new modeling.

## Boundary 1 — `revision-continuity` vs commit rate `cps`

A subscriber that reconnects must replay from within the rolling window `W` (plus checkpoint window).
The replay window covers `W / cps` seconds of history; as the commit rate `cps` climbs against a fixed
`W`, the window stops covering a subscriber's reconnect gap and continuity breaks (a gap appears).

- **axis** `cps` — commits/sec (monotone-in-harm at fixed `W`: more commits ⇒ shorter time coverage).
- **claim** `revision-continuity` — `holds(cps)` returns whether the replay window still covers a
  subscriber reconnect gap of the modeled duration: i.e. `coverageSeconds(W, cps) ≥ reconnectGapSec`
  (and/or `classifyResume` returns an in-window kind, not `OutOfWindow`, given that lag). Use the
  layered bound (`W + W_checkpoint`) consistent with reactivity.md and
  `reactivity-resume-classifier-layered-bound`.
- **designAssumption** the design's nominal commit rate (DEFAULT_REACTIVITY_CONFIG / the doc's worked
  example, e.g. 10 cps). Output: `cps*` at which coverage first drops below the reconnect gap,
  `margin = cps* − cps_nominal`; document that `W` (and the adaptive-W recommendation `ceil(60·cps)`)
  is what buys the margin.
- This axis is the pure-function path (no full sim) — cheap; no N-gating needed.

## Boundary 2 — tail-rotation drain vs `T_rejoin_jitter / T_drain` ratio

On tail rotation, all subscribers re-register at the new tail with jitter over `T_rejoin_jitter`; the
old tail keeps forwarding until `T_drain`. The re-registration wave must drain before the old tail
stops forwarding. `simulator-strengthen-scenario-adversariality` notes the current
`completes-within-drain` claim is near-tautological because `T_rejoin_jitter (30s) < T_drain (60s)` is
hardcoded — so the *ratio* is exactly the stress axis that gives the claim teeth.

- **axis** `rejoinDrainRatio = T_rejoin_jitter / T_drain` — sweep `T_rejoin_jitter` (or shrink
  `T_drain`) so the ratio climbs past 1 (monotone-in-harm: a wider rejoin spread vs the drain window
  ⇒ more likely the wave outlasts forwarding).
- **claim** `completes-within-drain` (drain dynamics, not just the arrival-window bound) —
  `holds(ratio)` runs `simulateRotationBurst` with that ratio and returns whether the wave drains
  (last re-registration absorbed, and — per the adversariality note — via promotion fan-out, not via
  the window being trivially short) before `T_drain`.
- **designAssumption** the shipped ratio `30s / 60s = 0.5`. Output: `ratio*` at which the wave first
  fails to drain, `margin = ratio* − 0.5`; this turns the tautological pass into a real margin.

## Edge cases & interactions

- **De-tautologize the drain claim.** The whole point of Boundary 2 is that the existing claim cannot
  be false at the shipped ratio. Drive the ratio past 1 so the claim *can* fail, and assert a positive
  control (a ratio known to outlast `T_drain`) flips it — otherwise the boundary is vacuous (same trap
  as the structural `heal-convergence` caveat).
- **Layered vs single resume bound.** Boundary 1 must use the layered bound (`W + W_checkpoint`)
  reactivity.md is authoritative on; the simulator's `classifyResume` historically used the single
  `W_checkpoint` bound (`reactivity-resume-classifier-layered-bound`). Use the corrected/layered path
  and note which bound the boundary assumes, so a reader knows whether `cps*` is the conservative or
  the layered edge.
- **`coverageSeconds` integer/zero edges.** `cps = 0` (silent topic) makes coverage unbounded — guard
  the axis floor above 0; very large `cps` must not overflow. The reconnect-gap duration must be a
  named constant, not implicit, since it sets where `cps*` lands.
- **Burst determinism.** `simulateRotationBurst` owns its own seeded world; the ratio axis must pass
  the seed through so two runs at the same `(seed, ratio)` give identical drain outcomes.
- **Fast-promote interaction.** The burst disables slope lookahead and relies on `cap_promote_fast`;
  the drain boundary must keep that wiring (don't accidentally re-enable lookahead, which would change
  the peak-root mechanics the claim depends on).
- **Cross-ref.** `simulator-strengthen-scenario-adversariality` (drain claim hardening),
  `reactivity-resume-classifier-layered-bound` (resume bound). Design as if both land.
