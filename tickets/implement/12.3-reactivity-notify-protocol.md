# Reactivity notify protocol surface + transport primitives

## Goal

Stand up the reactivity libp2p **protocol family** and the **one-way notification transport** (frame →
dial → deliver) that the rest of the 12.3 chain builds on. This ticket is pure transport plumbing: it
delivers `NotificationV1` bytes from a dialer to a registered handler, with **no fan-out orchestration,
no role decision, and no gossip** (those are siblings `reactivity-forwarder-host` and
`reactivity-pushstate-gossip`). Mirror the cohort-topic protocol/transport modules verbatim in style so
the family reads identically to `packages/db-p2p/src/cohort-topic/`.

The db-core `NotificationV1` codec already exists: `encodeNotificationV1` / `decodeNotificationV1`
(`packages/db-core/src/reactivity/wire.ts:193-200`, length-prefixed JSON, base64url byte fields). This
ticket only frames over libp2p.

## What to build

### `packages/db-p2p/src/reactivity/protocols.ts` (new) — mirror `cohort-topic/protocols.ts`

Define the reactivity protocol family under one base so the sibling tickets extend it without renaming:

```
REACTIVITY_BASE = "/optimystic/reactivity/1.0.0"
PROTOCOL_REACTIVITY_NOTIFY             = `${REACTIVITY_BASE}/notify`              // NotificationV1 fan-out (one-way)
PROTOCOL_REACTIVITY_PUSH_STATE_GOSSIP  = `${REACTIVITY_BASE}/push-state-gossip`  // PushStateGossipV1 (one-way)  [used by reactivity-pushstate-gossip]
// reserved (NOT defined here, named so the family is understood): `${REACTIVITY_BASE}/recover` (12.4 backfill/resume)

interface ReactivityProtocols { notify: string; pushStateGossip: string }
DEFAULT_REACTIVITY_PROTOCOLS: ReactivityProtocols
makeReactivityProtocols(networkName = "default"): ReactivityProtocols   // mirror makeCohortTopicProtocols namespacing
reactivityProtocolList(p: ReactivityProtocols): string[]                 // for node.handle / unhandle over the set
```

Follow `cohort-topic/protocols.ts` exactly: canonical network-agnostic ids omit the network segment;
`makeReactivityProtocols("default")` inserts `/optimystic/default/reactivity/1.0.0/...` (the FRET
`makeProtocols` convention). The push-state-gossip id is **declared here** (one place owns the family)
even though `reactivity-pushstate-gossip` consumes it.

### `packages/db-p2p/src/reactivity/notify-transport.ts` (new) — mirror `cohort-gossip-transport.ts`

A **unicast** one-way transport (cohort-gossip is broadcast; notify dials a single named target):

```
interface ReactivityNotifyTransport {
  /** Frame `n` (encodeNotificationV1) and dial `target` (peer-id string) over the notify protocol.
   *  Fire-and-forget, failure-isolated: a dead/unreachable target's rejection is swallowed (logged),
   *  never propagated to the caller's fan-out loop. */
  send(target: string, n: NotificationV1): Promise<void>;
  /** Subscribe to inbound notifications (after decode); returns an unsubscribe handle. */
  onNotification(handler: (from: PeerRef, n: NotificationV1) => void): () => void;
  /** Feed an inbound notify frame (called by the host's notify protocol handler). */
  deliver(fromPeerId: string, frame: Uint8Array): void;
}

class Libp2pReactivityNotifyTransport implements ReactivityNotifyTransport {
  constructor(node: Libp2p, options?: { notifyProtocol?: string; maxBytes?: number; selfPeerId?: string })
}
```

- `send`: reuse `sendOneWay(node, peerIdFromString(target), notifyProtocol, encodeNotificationV1(n))`
  from `stream-util.ts`; wrap in `.catch(() => { /* best-effort, logged */ })`. If `selfPeerId` is
  supplied and `target === selfPeerId`, skip the dial (caller may instead deliver in-process — the
  forwarder-host ticket decides; the transport just never dials self).
- `deliver`: `decodeNotificationV1(frame)` inside a try; on decode failure log + drop (a malformed
  frame must never throw out of a stream handler). Fan the decoded notification to every `onNotification`
  subscriber, building `from: PeerRef = { id: peerIdToBytes(fromPeerId) }` (mirror
  `FretCohortGossipTransport.deliver`).

### Inbound handler registration helper (this module)

```
registerNotifyHandler(node: Libp2p, protocol: string, transport: ReactivityNotifyTransport, maxBytes?): void
```

Mirror `stream-util.ts` `handleRequestResponse` but **one-way** (no reply frame): read one bounded frame,
call `transport.deliver(connection.remotePeer.toString(), frame)`, return nothing, close. Catch + abort on
error exactly like `makeFrameHandler` / `handleRequestResponse` (never throw on the stream). Use
`readAllBounded` with `DEFAULT_STREAM_MAX_BYTES`.

### Export from `packages/db-p2p/src/reactivity/index.ts`

Add `export * from "./protocols.js"` and `export * from "./notify-transport.js"`.

## Edge cases & interactions

- **Malformed / undecodable inbound frame** → dropped + logged, handler never throws (matches cohort-topic
  one-way handlers `host.ts:1403-1406`, `1412-1425`).
- **Oversized frame** → bounded by `readAllBounded(stream, maxBytes)`; over-bound read aborts the stream,
  no partial delivery.
- **Dead / unreachable target** → `send`'s `.catch` isolates it; one dead subscriber must not break a
  fan-out loop or a commit (the load-bearing failure-isolation property of §Propagation).
- **Self target** → never dialed (`selfPeerId` guard); the forwarder-host ticket handles in-process
  delivery for a subscriber co-located with the origin.
- **Concurrent inbound streams / concurrent sends** → the transport holds no per-call mutable state beyond
  the handler set; each `send` is an independent dial. No ordering guarantee (reactivity is hint-only;
  contiguity is the subscriber's job via `revision == lastRevision + 1`).
- **Protocol namespacing** → `makeReactivityProtocols("x")` must match the FRET `makeProtocols` shape
  (segment inserted even for `"default"`); the canonical const omits it.
- **No reply frame** → notify is strictly one-way; a handler that tried to `stream.send` a reply would
  desync the dialer's `sendOneWay` (which closes after send). Assert nothing is sent back.
