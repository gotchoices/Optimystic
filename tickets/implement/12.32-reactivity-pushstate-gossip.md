# Reactivity PushState gossip over the cohort bus

## Goal

Drive `PushState.serializeGossip()` / `mergeGossip()` over the wire on the cohort gossip cadence so the
replay ring + dedupe window converge across the whole forwarder cohort — making the doc's "any cohort
member can serve a replay/backfill" property real over libp2p, not just in-process. Today the codec
exists and is tested (`push-state.ts` `encode/decodePushStateGossipV1`, `serializeGossip`, `mergeGossip`)
but nothing broadcasts or merges it.

This is the **replication** backstop for the live fan-out (`reactivity-forwarder-host`): a member that
missed an origin dial still gets the entry via gossip and can serve a backfill (`12.4`) after a primary
failover.

## Design (resolved)

**Separate one-way protocol, broadcast over the existing cohort gossip transport.** Use
`PROTOCOL_REACTIVITY_PUSH_STATE_GOSSIP` (declared by `reactivity-notify-protocol`). The cohort gossip
transport already exposes the exact seam for this:
`FretCohortGossipTransport.broadcastOver(protocol, coord, msg)` — built so non-`/cohort-gossip` protocols
(it cites the promote-notice broadcast) reuse the cohort peer resolution + self-exclusion + per-peer
failure swallowing. So PushState gossip rides `broadcastOver(PROTOCOL_REACTIVITY_PUSH_STATE_GOSSIP,
cohortCoord, encodePushStateGossipV1(state.serializeGossip()))`, fanned to the FRET-assembled cohort
around the collection's served coord. Inbound frames arrive on this node's `push-state-gossip` handler
and route to the owning collection's `pushState.mergeGossip` (which already guards foreign
collection/topic).

**Cadence: own unref'd timer** (parallel to the cohort host's single `setInterval`, `host.ts:669`).
Rationale: the host's timer is private and ticks per-`CoordEngine`; reactivity collections are not 1:1
with coord engines, and the forwarder host owns the live `PushState` set. Reuse
`DEFAULT_GOSSIP_INTERVAL_MS` and the host's re-entrancy/`stopped`/`unref` discipline verbatim. Document
the tradeoff: a future consolidation could fold this into the host tick via a hook, but a separate
unref'd timer is the clean seam now and cannot pin an idle process.

## What to build: `packages/db-p2p/src/reactivity/push-state-gossip.ts` (new)

```
interface ReactivityPushStateGossipDriverDeps {
  /** Reuse the node's cohort gossip transport (broadcastOver) — no new transport. */
  gossipTransport: Pick<FretCohortGossipTransport, "broadcastOver">;
  /** The live forwarder PushStates to gossip each round, with the cohort coord to broadcast to. */
  liveCollections: () => Iterable<{ pushState: PushState; cohortCoord: RingCoord }>;
  /** Resolve the collection owning an inbound frame (by collectionId/topicId) → its PushState, or undefined. */
  pushStateForGossip: (g: PushStateGossipV1) => PushState | undefined;
  /** Inbound authenticity gate: is `fromPeerId` a member of the cohort for the frame's coord? */
  isCohortMember?: (fromPeerId: string, g: PushStateGossipV1) => boolean;
  intervalMs?: number;     // default DEFAULT_GOSSIP_INTERVAL_MS
  maxBytes?: number;       // default DEFAULT_STREAM_MAX_BYTES
  clock?: () => number;
}

class ReactivityPushStateGossipDriver {
  start(): void;                                   // begin the unref'd cadence timer
  /** One round: broadcast each live collection's serialized PushState (chunked to fit maxBytes). */
  round(): void;
  /** Inbound handler body (registerNotifyHandler-style): decode → gate → mergeGossip. */
  deliver(fromPeerId: string, frame: Uint8Array): void;
  stop(): void;                                    // clear timer
}
```

- `round()`: for each live collection, `g = pushState.serializeGossip()`; **bound the frame** (see edge
  cases) then `gossipTransport.broadcastOver(PROTOCOL_REACTIVITY_PUSH_STATE_GOSSIP, cohortCoord,
  encodePushStateGossipV1(g, maxBytes))`. Re-entrancy + `stopped` guard like the host tick.
- `deliver`: `decodePushStateGossipV1(frame)` (try/catch → drop+log); run `isCohortMember` gate (drop a
  non-member sender); `pushStateForGossip(g)?.mergeGossip(g)`.
- The node-wiring ticket registers the inbound handler (`registerNotifyHandler`-style over
  `protocols.pushStateGossip` → `driver.deliver`) and supplies `liveCollections` / `pushStateForGossip`
  from the `ReactivityForwarderHost`'s PushState map, `gossipTransport` from the host, and `isCohortMember`
  from FRET cohort assembly.

## Edge cases & interactions

- **Frame-size bound (load-bearing).** `serializeGossip()` serializes the **whole** replay ring; at
  `W = 256` full `NotificationV1` entries this can exceed `DEFAULT_STREAM_MAX_BYTES` (512 KiB). The driver
  MUST keep each frame within the bound: serialize, and if over-bound, send the **most-recent-N** replay
  entries that fit (slice `g.replayBuffer.entries` from the high-revision end; keep the full `dedupe`
  window — it is small). Convergence then completes over a few rounds. `mergeGossip` already unions ring
  entries (`replayBuffer.merge`) so a partial frame loses nothing. Log when a frame is truncated (no
  silent cap — the AGENTS.md "no silent caps" rule). Add a test that a buffer larger than the bound stays
  within `maxBytes` and converges over rounds.
- **Gossip for an unserved collection** → `pushStateForGossip` returns `undefined` → ignored; and
  `mergeGossip` independently guards `collectionId`/`topicId` mismatch.
- **Non-member / forged sender** → `isCohortMember` gate drops before merge. Defense-in-depth: every
  replay entry is a full end-to-end-verifiable `NotificationV1`, so a backfill *serve* re-verifies entries
  regardless — a poisoned ring entry cannot produce a forged delivery, only waste a slot. **Full peer-sig
  envelope signing of the push-state-gossip frame is a documented hardening follow-on** (file a
  `backlog/` ticket): the membership gate + self-verifying entries are the defensible default for this
  hint-only path; envelope signing mirrors cohort-topic's `peer-sig` gossip auth if/when needed.
- **Backpressure map never gossiped** — `PushStateGossipV1` deliberately omits `perSubscriberQueue`
  (primary-local). Confirm the driver only ever ships `serializeGossip()` output (it already excludes it).
- **Epoch / rotation churn** — a rotation rebuilds the tail `PushState` fresh; stale gossip for the old
  coord routes to no live collection (or a different coord) and is ignored. (Rotation lifecycle is 12.5.)
- **Timer discipline** — re-entrancy guard (skip overlapping round), `stopped` short-circuit, `unref()`
  so gossip never pins an idle process; `stop()` clears the timer (node-wiring ticket also unhandles the
  protocol).
- **Convergence target** — one round after an origin fan-out, a member that missed the dial holds the
  entry and can serve the same backfill the origin would (the any-member-serves-replay property).
