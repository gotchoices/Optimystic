# Cohort-topic host + change-bridge: production node wiring (activation)

The `local-change-notifier-bridge` ticket (now complete) shipped the full origination **mechanism**
and the two cluster/notifier **seams**, fully unit-tested, but deliberately left it **dormant in the
production node**. This ticket flips activation on: construct the cohort host on the running node and
install the bridge so a commit landing on a cohort-member node actually originates into
`CohortTopicService.onLocalCommit`.

## Why this is a separate ticket

The cohort host is constructed **post**-node (`createCohortTopicHost(node, fret, …)` consumes an
already-assembled libp2p node + FRET), so activation cannot live inside `createLibp2pNodeBase`. The
bridge ticket therefore shipped exactly the seam and explicitly deferred whole-node wiring to "the
cohort-host node-wiring / live-tier work" — this ticket. The real-libp2p e2e suite
(`14-substrate-e2e-real-libp2p-tier`) needs origination live on production nodes, so this must land
before that suite can exercise reactivity end-to-end.

## What's already in place (do not re-build)

- `StorageRepo.onAnyCollectionChange(listener)` — catch-all commit feed.
- `makeCohortTopicChangeNotifier(deps)` / `attachCohortChangeBridge(node, deps)` — the bridge + the
  post-assembly installer that sets `node.blockChangeNotifier`.
- `ClusterMember.onCommitCertificate` sink (fired BEFORE `storageRepo.commit`) + the
  `onCommitCertificate?` node option on `NodeOptions`, already threaded to `clusterMember(...)`.
- `createCommitCertStore()` (bounded TTL cache) + `makeClusterCommitCertExtractor(store)`.

## What this ticket must add

The whole-node wiring, on a node that runs the cohort-topic substrate:

```
const store = createCommitCertStore();
// cluster side: capture the consensus cert into the store, keyed by actionId, before commit applies
createLibp2pNodeBase({ …, onCommitCertificate: store.put });
// …after the node + FRET are up, construct the cohort host:
const service = await createCohortTopicHost(node, fret, …);
// notifier side: install the bridge as node.blockChangeNotifier
attachCohortChangeBridge(node, {
  source: storageRepo,                                  // the node's StorageRepo (ChangeBridgeSource)
  service,
  selfIsCohortMember,                                   // SEE BELOW — the concrete predicate
  extractCommitCert: makeClusterCommitCertExtractor(store),
});
```

### Open design points to resolve in this ticket

- **`selfIsCohortMember(collectionId)` concrete impl.** The bridge is membership-gate agnostic; this
  ticket supplies the real predicate: derive the collection's reactivity topic
  (`topicId = H(tailId ‖ "reactivity")` — the helper defined by `reactivity-origination-replay-delivery`;
  reuse it, do not re-derive) and consult the cohort host registry / FRET membership for that topic.
  Resolve how the bridge obtains `tailId` for a `collectionId` at emit time (the change event carries
  `blockIds` + `collectionId`, not the tail) — likely via the host's per-collection reactivity state.
- **Ordering / `blockChangeNotifier` capture.** `attachCohortChangeBridge` mutates
  `node.blockChangeNotifier` after assembly, but `libp2p-node-base.ts:637` already sets it to the bare
  `storageRepo`, and the Quereus `collection-factory` **caches** the notifier reference at first
  collection creation. Confirm the bridge is attached before any collection/transactor captures the
  notifier, or that late attachment is acceptable (the bridge delegates `onCollectionChange` to the
  same `source`, so reactive-watch keeps working either way — but the *origination* path only runs for
  collections created after attach unless attach happens during assembly).
- **`attachCohortChangeBridge` node typing.** It types `node` as
  `{ blockChangeNotifier?: IBlockChangeNotifier }`, while the node exposes the property via
  `(node as any).blockChangeNotifier`. Tidy the cast at the call site (or widen the node type) so the
  wiring is type-clean.
- **Bridge / catch-all subscription teardown.** `makeCohortTopicChangeNotifier` subscribes to the
  catch-all feed once and never unsubscribes (node-lifetime by intent). Decide whether node shutdown
  needs a teardown handle (e.g. when the cohort host withdraws / the node stops) or whether
  node-lifetime is genuinely sufficient; if a handle is wanted, thread an unsubscribe out of
  `attachCohortChangeBridge`.
- **Activation gating.** Only wire when the node actually runs the cohort-topic substrate (the host is
  optional / feature-gated). A node with no host must keep today's behavior (bare `storageRepo` as
  `blockChangeNotifier`, zero cohort cost).

## Done-when

- A production libp2p node that runs the cohort-topic host originates `onLocalCommit(event, cert)` for
  member commits, with the commit cert resolved from the live `CommitCertStore` (no test-only shims).
- `selfIsCohortMember` consults real cohort/FRET membership for the collection's reactivity topic.
- `docs/internals.md` "Wiring status" bullet updated: origination is live in the production node, not
  dormant.
- Build + tests green for db-p2p; the change is exercised by the reactivity / real-libp2p e2e tiers.
